## Description

Implement the authenticated REST API for creating, listing, retrieving, updating, and deactivating short links. All routes require a valid session. URL filtering and deduplication are integrated into POST (see #0023, #0024).

## Acceptance Criteria

- [ ] `POST /api/links`: body `{destination_url, title?, key?, expires_at?}`; applies filter rules (#0024) then dedup (#0023); returns link object with `"duplicate": true/false`; returns `422` if blocked by filter
- [ ] `GET /api/links`: returns paginated list of authenticated user's links; supports `?page=` and `?per_page=` (default 20); each item includes `{id, key, destination_url, title, active, denied_reason, created_at, expires_at, click_count}`
- [ ] `GET /api/links/{key}`: returns full link detail including click stats; returns `404` if not owned by authenticated user
- [ ] `PATCH /api/links/{key}`: accepts partial `{title?, destination_url?, expires_at?, active?}`; updates only provided fields; returns updated link; returns `404` if not owned by authenticated user
- [ ] `DELETE /api/links/{key}`: sets `active = false`; evicts key from redirect cache; writes `link.deactivated` audit entry; returns `200`
- [ ] All routes require auth (use `RequireAuth` middleware from #0017)
- [ ] `go test ./internal/handlers/` covers all five routes including ownership checks

## Notes

Dependencies: #0006 (links table), #0007 (cache eviction on delete), #0017 (auth middleware), #0023 (dedup), #0024 (filter), #0025 (audit).
