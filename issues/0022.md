## Description

Implement the authenticated REST API for creating, listing, retrieving, updating, and deactivating short links. All routes require a valid session. URL filtering and deduplication are integrated into POST (see #0023, #0024).

## Acceptance Criteria

- [x] `POST /api/links`: body `{destination_url, title?, key?, expires_at?}`; returns link object with `"duplicate": true/false` (always `false` for now); 201. Filter (`422`, #0024) and dedup (#0023) are left as ordered seams — NOT in scope here.
- [x] `GET /api/links`: returns paginated list of authenticated user's links; supports `?page=` and `?per_page=` (default 20); each item includes `{id, key, destination_url, title, active, denied_reason, created_at, expires_at, click_count}`
- [x] `GET /api/links/{key}`: returns full link detail including click stats; returns `404` if not owned by authenticated user
- [x] `PATCH /api/links/{key}`: accepts partial `{title?, destination_url?, expires_at?}`; updates only provided fields; returns updated link; returns `404` if not owned by authenticated user
- [x] `DELETE /api/links/{key}`: sets `active = false`; evicts key from redirect cache (seam — no cache wired yet); returns `200`. `link.deactivated` audit is a seam for #0025.
- [x] All routes require auth (mounted behind `middleware.RequireSession` from #0017)
- [x] `go test ./internal/handlers/` covers all five routes including ownership checks

## Implementation

Added a DB-backed links data layer and the authenticated `/api/links` endpoints, all behind `RequireSession` and scoped to the authenticated user (`user_id = ctx user`, enforced in SQL).

**`internal/links/store.go`** — new `links.Store` over the shared pgx pool:

- `KeyExists` — `SELECT EXISTS(... WHERE key=$1)`; backs the `GenerateUniqueKey` collision closure (key is globally UNIQUE, so not user-scoped).
- `CreateLink(NewLink)` — inserts `active=true, denied_reason=0, created_at=now()`; maps a `23505` unique violation on `key` to `ErrKeyTaken` (→ 409 for a clashing custom alias).
- `ListLinks(userID, limit, offset)` — most-recent-first (`created_at DESC, id DESC`), one `LEFT JOIN clicks ... GROUP BY` so `click_count` comes back in the same query; `CountLinks` for pagination total.
- `GetLink(userID, key)` — detail + click count via a correlated subquery; `ErrLinkNotFound` when absent OR owned by another user (indistinguishable, no leak).
- `UpdateLink(userID, key, LinkUpdate)` — dynamic SET of only the provided `title`/`destination_url`/`expires_at`; re-reads the row; `ErrLinkNotFound` for foreign keys.
- `DeactivateLink(userID, key)` — soft delete (`active=false`, row retained); `ErrLinkNotFound` when zero rows affected.

**`internal/handlers/links.go`** — `LinksHandler` (Create/List/Get/Patch/Delete) over a `linkStore` interface and an optional `cacheEvictor` (nil = skip eviction). Syntactic absolute-http(s)-URL validation (`validDestinationURL`), custom-alias validation (`validKey`, 1–12 url-safe chars), `?page=`/`?per_page=` parsing (default 20, cap 100). `patchLinkRequest` uses a custom `UnmarshalJSON` so `expires_at: null` clears the column while an omitted key leaves it unchanged.

**`cmd/shortlinks/main.go`** — constructs `links.NewStore(pool)` + `handlers.NewLinksHandler(store, nil)` and mounts all five routes behind `requireSession`. Cache passed as `nil` with a TODO (no redirect-cache instance lives in `serve()` yet).

## Gotchas

Ordered seams left for the layered issues (search for `SEAM #00xx` in the code):

- **#0024 (URL filter)** — `internal/handlers/links.go`, `LinksHandler.Create`: marked `SEAM #0024 (URL filter check — runs FIRST)`, between body validation and key resolution. Runs before dedup; on a match insert a denied link and return `422`.
- **#0023 (dedup)** — same method, `SEAM #0023 (dedup check — runs AFTER the filter, before insert)`, inside the generated-key branch only (custom aliases are not deduplicated). Flip the local `duplicate := false` to `true` on a hit and skip `GenerateUniqueKey`/insert. `links.Store` exposes `Pool()` for an atomic dedup-or-insert tx.
- **#0025 (audit)** — `Create` (after a successful insert: `link.created` with `{key, destination_url, title, duplicate}`) and `Delete` (`link.deactivated` with `{key, destination_url}`); both marked `SEAM #0025`.
- **#0026 (SSE)** — `Create`, marked alongside the #0025 audit seam: `broker.Publish(u.ID, Event{Name:"link.created", ...})` after create/reactivate.
- **Redirect cache eviction** — `LinksHandler` takes an optional `cacheEvictor` and calls `h.evict(key)` on PATCH and DELETE, but `serve()` passes `nil` because no redirect-cache instance is constructed there yet (the `GET /u/{key}` route is wired in a separate issue).
