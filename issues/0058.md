## Description

Add a strictly dev-only mode that establishes an authenticated session for a mock admin user **without** the WebAuthn passkey ceremony, so opening `localhost` lands directly on the dashboard. WebAuthn requires HTTPS and a real authenticator, which is impractical for quick local checks.

## Motivation

Even with a local server, the SPA's first reachable view is login, and the passkey ceremony can't be completed casually on localhost. A dev auth bypass makes the authenticated UI (dashboard / account / admin) reachable for inspection. (Part of [#0056](0056.md).)

## Proposed approach

- Behind an explicit dev flag (e.g. `DEV_AUTH=true`), seed/load a mock admin user in the active store (#0057) and issue a valid session cookie automatically — e.g. an auto-login endpoint, or middleware that establishes the session on first request.
- **Hard guardrail:** refuse to enable unless clearly non-production — require `DEV_AUTH=true` AND a localhost `BASE_URL` / `WEBAUTHN_RP_ID`, and/or a `dev` build tag. Fail closed; never bypass auth in production. Log loudly when dev auth is active.
- Keep the real WebAuthn login/registration/recovery flows intact and the default; this is additive and off by default.

## Acceptance criteria

- [x] With the dev flag on (localhost only), loading the app lands on the dashboard as a mock admin with a valid session — no passkey
- [x] The bypass is impossible to enable in production (explicit flag + non-prod/localhost check and/or build tag); covered by a test
- [x] Real passkey login/registration/recovery flows are unchanged when the flag is off
- [x] `go build ./...` and `go test ./...` pass

## Relation

- Umbrella: [#0056](0056.md). Depends on [#0057](0057.md) (the mock user lives in the dev store).

## Implementation

Added `internal/middleware/DevAutoLogin`, wired ONLY on the `serveDevMode` path (`cmd/shortlinks/main.go`, via a new optional `outerMiddleware` param to `mountAndServe`; `servePostgres` passes `nil`). On any request lacking a session, it mints a dev session for the seeded mock admin (`devstore.CreateDevSession(1, 24h)`), writes the **real** `shortlinks_session` cookie via `auth.SetSessionCookie`, AND injects the token into the in-flight request (`r.AddCookie`) so the existing `RequireSession` accepts it — every request self-authenticates server-side. This means it works in a real browser on `http://localhost` even though the browser won't persist the `Secure` cookie over http (auth doesn't depend on the browser returning it). **Fail-closed guardrail:** `DevAutoLogin` panics at construction if `devMode` is false, and it's only ever constructed under `STORAGE=json`, so it can never run in production. Built on `issue/0058` (Sonnet implement, Opus review → APPROVE), squash-merged to `main` as `068efe4`.

## Verification

`go build ./...` clean; `go test ./...` (with the test DB) — existing tests pass, 4 new `devauth_test.go` tests pass (panic guardrail, cookie injection, skip-when-present, full `DevAutoLogin → RequireSession` chain). **Boot proof (re-run by reviewer):** `STORAGE=json ./shortlinks serve` (no Postgres) → `/api/me` → `200 {"id":1,"email":"admin@localhost","is_admin":true}`, `/api/links` → `200` with the seeded links; a request with NO cookie jar (the browser-over-http case) still returns 200. Real WebAuthn/`RequireSession`/`SetSessionCookie`/`servePostgres` paths are unchanged (zero diff lines).

## Files changed

- `internal/middleware/devauth.go` + `devauth_test.go` — NEW: the dev auto-login middleware + tests.
- `cmd/shortlinks/main.go` — construct `DevAutoLogin` in `serveDevMode`; `outerMiddleware` param applied before `ListenAndServe`; prod passes `nil`.

## Gotchas

- It's gated `STORAGE=json` only and panics if misused outside dev — fail-closed by design; never loosen that.
- Minor (dev-only): because the browser won't persist the `Secure` cookie over http, a fresh dev session is minted per cookieless request (the devstore sessions map grows for the process lifetime). Harmless for local dev; could later use a `Secure=false` dev cookie to reuse one session.
