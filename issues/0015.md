## Description

Implement the three-step passkey registration flow: email submission, magic-link verification, and WebAuthn attestation completion. iCloud Keychain compatible (authenticatorAttachment omitted). First registered email matching ADMIN_EMAIL is promoted to admin.

## Acceptance Criteria

- [ ] `POST /auth/register/start`: checks `registrations_enabled` setting; returns `403` if disabled; validates email not already registered; creates `pending_registrations` row (5-min TTL); sends verification email via `Mailer`; returns `200 {"message":"Check your email"}`
- [ ] `GET /auth/register/verify?token=...`: validates token exists and not expired; issues WebAuthn `PublicKeyCredentialCreationOptions` with `residentKey: "required"`, `userVerification: "required"`, `authenticatorAttachment` omitted; stores challenge in `webauthn_challenges` linked to `pending_registration_token`; returns options JSON
- [ ] `POST /auth/register/finish`: verifies challenge; stores `passkey_credentials` row; creates `users` row with `active=true`; if no other users exist OR email matches `ADMIN_EMAIL`, sets `is_admin=true`; deletes `pending_registrations` row; creates session; sets `HttpOnly; Secure; SameSite=Strict` cookie; returns `200`
- [ ] Uses `github.com/go-webauthn/webauthn` library
- [ ] `pubKeyCredParams`: ES256 (`-7`) preferred, RS256 (`-257`) accepted
- [ ] `user.id` in WebAuthn options is a random 16-byte value (not derived from email)
- [ ] Writes `account.registration_started` audit entry on start, `account.registered` on finish
- [ ] Writes `credential.added` audit entry with `{device_name, aaguid}` metadata
- [ ] `go test ./internal/auth/` and `go test ./internal/handlers/` cover the happy path and error cases (duplicate email, expired token, invalid challenge)
