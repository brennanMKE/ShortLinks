## Description

Write a `DEPLOYMENT.md` at the project root covering the full end-to-end process to deploy ShortLinks on a fresh EC2 instance. This document is used by the initial operator and by anyone cloning the repo to run their own server.

## Acceptance Criteria

- [ ] `DEPLOYMENT.md` covers all of the following sections in order:
  1. **Prerequisites** — EC2 instance (Ubuntu 22.04+), Apache 2 installed, PostgreSQL installed, Node.js 20+ and Go 1.22+ installed, DNS wildcard pointing to EC2
  2. **Database setup** — run `scripts/db/create.sql`, configure password
  3. **Configuration** — copy `.env.example` to `/etc/shortlinks/config.env`, fill in all values (with notes on generating `SESSION_SECRET` via `openssl rand -hex 32`)
  4. **Build** — `cd web && npm ci && npm run build`, then `go build -o shortlinks ./cmd/shortlinks`
  5. **Migrations** — `migrate -path migrations -database $DATABASE_URL up`
  6. **Seed** — run seed command to create admin user and test link
  7. **systemd** — install unit file, enable and start service
  8. **Apache** — install virtual host config, enable site, reload Apache
  9. **TLS** — obtain Let's Encrypt certificate with Certbot
  10. **First login** — register account at `https://go.sstools.co` using the `ADMIN_EMAIL` address
- [ ] Each step includes the exact commands to run
- [ ] A **Updating** section covers the redeploy flow (rebuild, restart service, run new migrations)
