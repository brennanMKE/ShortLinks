# 0051 — Support light and dark mode across the SPA

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-18 |

## Description

Make the SPA adapt to light and dark mode. There is currently no theming infrastructure — no global stylesheet, no CSS custom properties, and no `prefers-color-scheme` handling; every component carries inline `<style>` with hardcoded hex colors (e.g. `#1f6feb`, `#c0362c`, `#666`). Introduce a small set of theme tokens and make the UI honor the OS color scheme.

## Motivation

A hardcoded light palette looks broken in dark-mode contexts (especially on phones at night) and is inconsistent across components. Centralizing colors also makes the responsive work and future visual changes far easier.

## Proposed approach

- Define a palette as CSS custom properties (`--color-bg`, `--color-surface`, `--color-text`, `--color-muted`, `--color-accent`, `--color-danger`, `--color-border`, …) on `:root`, with a dark override under `@media (prefers-color-scheme: dark)` (and/or a `[data-theme]` attribute if an explicit toggle is wanted).
- Introduce one global stylesheet (the SPA has none today) for the tokens and base element styling; migrate the hardcoded hex values in each component's inline `<style>` to the variables.
- Decide whether to ship an explicit light/dark/system toggle (persisted to `localStorage`) or follow the OS only — recommend OS-follow first, toggle as a possible follow-up.
- Set `<meta name="color-scheme">` / the CSS `color-scheme` property so form controls and scrollbars theme correctly.
- Verify contrast (WCAG AA) for text and interactive elements in both themes, including the error/notice colors.

## Acceptance criteria

- [ ] Theme tokens defined as CSS variables with a `prefers-color-scheme: dark` override
- [ ] All views legible and on-brand in both light and dark mode (no hardcoded hex left that breaks in dark)
- [ ] Native form controls/scrollbars respect the active scheme
- [ ] Text + interactive contrast meets WCAG AA in both themes
- [ ] `cd web && npm run build` and `npm run check` clean
