## Description

Wire the Svelte production build into the Go binary using `//go:embed`. The catch-all `GET /*` handler serves the SPA's `index.html` for all paths not matched by an API or redirect route, enabling hard-refresh support.

## Acceptance Criteria

- [x] `web/dist/` is populated by `npm run build` (run inside `web/`)
- [x] An embed source file contains a `//go:embed` directive over the build output.
  Placed at `web/embed.go` (`package web`) rather than `cmd/shortlinks/static.go`,
  because `//go:embed` paths are relative to the embedding file and cannot use `..`:
  ```go
  //go:embed all:dist
  var distFS embed.FS
  ```
- [x] `internal/handlers/static.go` serves the embedded FS:
  - Exact path matches (hashed assets like `assets/index-abc123.js`) served directly via `http.FileServerFS`
  - All other paths (`/`, `/anything`) return `index.html` (SPA fallback)
- [x] Static handler is the final route registered — after all `/u/`, `/api/`, `/auth/`, `/account/`, `/admin/`, `/health` routes
- [x] `go build ./cmd/shortlinks` succeeds with `web/dist/` present; binary runs and serves the SPA
- [x] `web/dist/` build artifacts are gitignored (`web/dist/*`), with only the
  minimal `web/dist/index.html` placeholder committed so a clean checkout compiles
- [x] The two-step production build (`cd web && npm run build && cd .. && go build ./cmd/shortlinks`)
  is documented in `issues/Issues.md` ("Build / verify command") and `PRD.md` ("Deployment")

## Implementation

- **Embed** — `web/embed.go` (`package web`) declares `//go:embed all:dist var distFS embed.FS`
  and exposes `DistFS() fs.FS`, which returns `fs.Sub(distFS, "dist")` so callers see
  `index.html` and `assets/*` at the FS root. The embed file lives in `web/` (not
  `cmd/shortlinks/`) because `//go:embed` paths are relative to the embedding file and
  cannot use `..` — `cmd/shortlinks` cannot reach `web/dist` without `../../`. The `all:`
  prefix includes dotfiles/underscore files that Vite may emit.
- **Clean-build guard** — `.gitignore` now ignores `web/dist/*` but allows
  `!web/dist/index.html`, and a minimal stub `web/dist/index.html`
  (`<!doctype html><title>ShortLinks</title><div id="app"></div>`) is committed. This keeps
  `//go:embed all:dist` (and therefore `go build ./...`) compiling on a clean checkout
  before any `npm run build`. The real build overwrites `index.html` and emits the hashed
  `dist/assets/*`, which remain ignored and are embedded only into the production binary.
- **SPA handler** — `internal/handlers/static.go` defines `SPAHandler` (constructed with
  `NewSPAHandler(fs.FS)`). For a request path that resolves to a real embedded file it
  delegates to `http.FileServerFS` (correct content types for hashed JS/CSS); for the root
  and any unmatched path (including directories) it writes `index.html` with
  `Content-Type: text/html; charset=utf-8`, so SPA deep links survive a hard refresh.
- **Wiring** — `cmd/shortlinks/main.go` `serve()` mounts `mux.Handle("GET /", handlers.NewSPAHandler(web.DistFS()))`
  as the final, least-specific route. Under the Go 1.22 `ServeMux` the explicit patterns
  (`/health`, `/u/{key}`, `/api/*`, `/auth/*`, `/account/*`, `/admin/*`) are more specific
  and always win, so the catch-all never shadows them.

## Files changed

- `web/embed.go` — new `package web`; `//go:embed all:dist` + `DistFS()` helper returning the `dist`-rooted `fs.FS`.
- `internal/handlers/static.go` — new `SPAHandler` serving embedded files with index.html SPA fallback.

## Gotchas

- **`//go:embed` cannot use `..`** and is relative to the embedding source file, so the embed
  file MUST live inside `web/` (not `cmd/shortlinks/`). This is why the embed package is `web`
  rather than the embed living next to `main.go` as the original acceptance criteria sketched.
- **`//go:embed dist` fails to compile if `dist` is empty**, and `web/dist/` is gitignored — so
  a naive setup breaks `go build` on a clean checkout / CI before anyone runs `npm run build`.
  The fix is the committed `web/dist/index.html` placeholder plus the `web/dist/*` +
  `!web/dist/index.html` ignore pattern; the real `npm run build` overwrites the placeholder and
  the hashed `assets/*` stay ignored.
- **Go 1.22 `ServeMux` matches the most specific pattern**, so the `GET /` catch-all is the
  least specific and never shadows the explicit API/redirect/health routes — confirmed live
  (`/health` still JSON, `/u/{key}` still 302). No registration-order tricks are required.
