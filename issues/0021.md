# 0021 — Registration settings toggle — admin control via settings table

## Description

Implement the server-side reading and enforcement of the `registrations_enabled` setting. The setting is read from the DB on each registration attempt (not cached) so admin changes take effect immediately without a restart.

## Acceptance Criteria

- [ ] `POST /auth/register/start` reads `registrations_enabled` from `settings` table on every call; returns `403 {"error":"registration_closed"}` when value is `"false"`
- [ ] `GET /admin/settings`: returns all rows from the `settings` table as `{"settings": [{"key": ..., "value": ...}]}`; requires admin auth
- [ ] `PATCH /admin/settings`: accepts `{"key": "registrations_enabled", "value": "true"}`; updates `settings` table; writes `settings.updated` audit entry with `{key, old_value, new_value}` metadata; returns `200`
- [ ] Only keys that already exist in the `settings` table can be updated (no arbitrary key creation via this endpoint); unknown keys return `400`
- [ ] `go test ./internal/handlers/` covers: registration blocked when disabled, registration allowed when enabled, admin settings read/write, non-admin blocked from settings routes

Dependencies: #0013 (settings table seeded with `registrations_enabled = false`), #0015 (register/start calls this check), #0017 (admin auth middleware).
