## Description

Create the systemd unit file that manages the ShortLinks service on the EC2 instance. The service must restart automatically on failure and load configuration from `/etc/shortlinks/config.env`.

## Acceptance Criteria

- [ ] Unit file at `deploy/systemd/shortlinks.service`
- [ ] `EnvironmentFile=/etc/shortlinks/config.env` loads all env vars
- [ ] `ExecStart` points to the installed binary (e.g. `/usr/local/bin/shortlinks`)
- [ ] `Restart=on-failure` with `RestartSec=5`
- [ ] Runs as a dedicated `shortlinks` system user (non-root)
- [ ] `WantedBy=multi-user.target`
- [ ] `deploy/systemd/README.md` documents:
  - Create system user: `sudo useradd --system --no-create-home shortlinks`
  - Install: `sudo cp shortlinks.service /etc/systemd/system/ && sudo systemctl daemon-reload && sudo systemctl enable shortlinks && sudo systemctl start shortlinks`
  - View logs: `sudo journalctl -u shortlinks -f`

## Notes

Dependencies: #0004 (deployment doc references this).

## Implementation

Added `deploy/systemd/shortlinks.service`, a `Type=simple` unit:

- `[Unit]`: `Description`, `Documentation`, `After=network.target postgresql.service`,
  `Wants=postgresql.service` (the service needs the local PostgreSQL up first).
- `[Service]`: `ExecStart=/usr/local/bin/shortlinks serve` (the install path from
  DEPLOYMENT.md step 4 and the `serve` subcommand from `cmd/shortlinks/main.go`),
  `EnvironmentFile=/etc/shortlinks/config.env`, dedicated non-root
  `User=shortlinks`/`Group=shortlinks`, `Restart=on-failure`, `RestartSec=5`, plus
  readable hardening directives (`NoNewPrivileges`, `ProtectSystem=strict`,
  `PrivateTmp`, `RestrictAddressFamilies`, etc.).
- `[Install]`: `WantedBy=multi-user.target`.

Added `deploy/systemd/README.md` documenting the system-user creation, the
install/enable/start command, and `journalctl` log viewing, exactly as the
acceptance criteria require.

## Verification

systemd is a Linux init system and is not present on this macOS host, so the unit
could not be loaded or started by a live systemd, and `systemd-analyze verify` is
unavailable here. The unit was verified by inspection:

- All three sections (`[Unit]`, `[Service]`, `[Install]`) present and well-formed;
  directive names spelled per the systemd unit syntax.
- `ExecStart` path (`/usr/local/bin/shortlinks`) matches DEPLOYMENT.md step 4
  (`sudo install -m 0755 shortlinks /usr/local/bin/shortlinks`) and the systemd
  section's forward reference.
- `EnvironmentFile=/etc/shortlinks/config.env` matches DEPLOYMENT.md step 3 and is
  the file populated from `.env.example`.
- The `serve` verb exists in `cmd/shortlinks/main.go` (dispatched in the `switch`
  to start the HTTP server).
- `go build ./...` passes, confirming the Go build is unaffected.
