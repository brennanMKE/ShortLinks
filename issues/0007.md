## Description

Implement `internal/cache/` wrapping `github.com/dgraph-io/ristretto` to cache short-link key → link row lookups. This is the primary performance optimisation for the redirect path.

## Acceptance Criteria

- [ ] `Cache` struct with `Get(key string) (*CachedLink, bool)`, `Set(key string, link *CachedLink)`, `Delete(key string)` methods
- [ ] `CachedLink` holds `DestinationURL`, `Active`, `ExpiresAt`, `DeniedReason`
- [ ] TTL read from config (`CACHE_TTL_SECONDS`, default 300s)
- [ ] Max entries read from config (`CACHE_MAX_COST`, default 10000)
- [ ] Negative caching: `SetNegative(key string)` stores a nil/sentinel entry with a 30-second TTL to prevent repeated DB hits for invalid keys
- [ ] Cache invalidation: `Delete` is called when a link is deactivated or denied
- [ ] Separate cache instance for URL filter rules with 60-second TTL (used by #0023)
- [ ] `go test ./internal/cache/` passes for get/set/delete/negative/TTL-expiry cases

## Implementation

Built `internal/cache` on top of `github.com/dgraph-io/ristretto/v2`
(see Gotchas for the v2 import-path note). The package is deliberately
self-contained — it does **not** import the `links` package — so the
domain layer can depend on cache later without a cycle.

- `CachedLink` is the cached value: `DestinationURL`, `Active`,
  `ExpiresAt *time.Time` (nullable), `DeniedReason int16`, and a
  `Negative bool` flag distinguishing a known-absent key from a real hit.
- `Cache` wraps a generic `ristretto.Cache[string, *CachedLink]`.
- `New(maxCost int64, ttl time.Duration)` builds the cache; non-positive
  args fall back to `DefaultMaxCost` (10000) and `DefaultTTL` (300s),
  matching `CACHE_MAX_COST` / `CACHE_TTL_SECONDS`. NumCounters is sized
  at 10x MaxCost per Ristretto guidance; each entry costs 1 unit so
  MaxCost behaves as a plain entry count.
- Methods: `Get(key) (*CachedLink, bool)`, `Set(key, *CachedLink)` (TTL),
  `SetNegative(key)` (fixed 30s `NegativeTTL`, value flagged `Negative`),
  `Delete(key)` for invalidation on deactivate/deny, plus `Wait()` and
  `Close()`.

Config wiring and the separate 60s filter-rule cache instance (criteria
items for `CACHE_TTL_SECONDS` config read and #0023) belong to the
consumers (config loader / links / filters) and are out of scope for
this self-contained cache package; the constructor accepts the TTL and
max-cost values those callers will supply.

## Gotchas

- **Ristretto `Set` is asynchronous.** `Set`/`SetWithTTL` enqueue the
  write to a buffer; a `Get` issued immediately afterward can miss. The
  package exposes `Wait()` (delegating to ristretto's `Wait()`) and the
  tests call it after every write — without this the tests flake.
- **Import path is `github.com/dgraph-io/ristretto/v2`, not the bare
  module.** `go get github.com/dgraph-io/ristretto` resolves to the old
  `v0.2.0` (untyped `interface{}` values); the maintained module is the
  separate major-version path `.../ristretto/v2` (resolved v2.4.0), which
  is generic — `Cache[K, V]` / `Config[K, V]` / `NewCache[K, V]`. We use
  v2 for the type safety of `Cache[string, *CachedLink]`.
