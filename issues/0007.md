## Description

Implement `internal/cache/` wrapping `github.com/dgraph-io/ristretto` to cache short-link key → link row lookups. This is the primary performance optimisation for the redirect path.

## Acceptance Criteria

- [ ] `Cache` struct with `Get(key string) (*CachedLink, bool)`, `Set(key string, link *CachedLink)`, `Delete(key string)` methods
- [ ] `CachedLink` holds `DestinationURL`, `Active`, `ExpiresAt`, `DeniedReason`
- [ ] TTL read from config (`CACHE_TTL_SECONDS`, default 300s)
- [ ] Max entries read from config (`CACHE_MAX_COST`, default 10000)
- [ ] Negative caching: `SetNegative(key string)` stores a nil/sentinel entry with a 30-second TTL to prevent repeated DB hits for invalid keys
- [ ] Cache invalidation: `Delete` is called when a link is deactivated or denied
- [ ] Separate cache instance for URL filter rules with 60-second TTL (used by #0023)
- [ ] `go test ./internal/cache/` passes for get/set/delete/negative/TTL-expiry cases
