## Description

Implement the `seed` subcommand (`go run ./cmd/shortlinks seed`) that creates the initial admin user from `ADMIN_EMAIL` and a test link pointing to `https://www.wikipedia.org`. The command must be idempotent — safe to re-run without creating duplicates.

## Acceptance Criteria

- [x] `go run ./cmd/shortlinks seed` (or compiled `shortlinks seed`) runs without error
- [x] Creates a `users` row with `email = ADMIN_EMAIL`, `is_admin = true`, `active = true` — skips if the email already exists
- [x] Creates a `links` row with a generated key, `destination_url = https://www.wikipedia.org`, `user_id` = the admin user's id, `active = true` — skips if a link with that destination already exists for the admin
- [x] Prints the generated short URL: `Seed link: https://go.sstools.co/u/{key}`
- [x] Idempotent: running twice produces no error and no duplicate rows
- [x] Command reads config from `.env` / environment (uses the same config loader as the server)

## Implementation

Added a `seed` subcommand alongside the existing `serve` case in
`cmd/shortlinks/main.go`. The logic lives in `cmd/shortlinks/seed.go`:

- Loads config via `config.Load()` and connects the pool via `db.Connect`.
- `ensureAdminUser` runs `INSERT ... ON CONFLICT (email) DO NOTHING` then an
  `UPDATE ... SET is_admin = TRUE, active = TRUE ... RETURNING id`, so a
  pre-existing (or non-admin) row is reused and converged to an active admin.
  The email is lowercased and trimmed before use.
- `ensureTestLink` first looks for an existing non-denied link
  (`denied_reason = 0`, matching the `idx_links_user_destination` partial
  index) to `https://www.wikipedia.org` for the admin and reuses its key.
  Otherwise it generates a unique 6-char key via `links.GenerateUniqueKey`,
  passing a closure backed by `SELECT EXISTS(SELECT 1 FROM links WHERE key=$1)`,
  and inserts the link with `active = true`, `denied_reason = 0`.
- Prints `Seed admin: <email> (id=N)` and the required
  `Seed link: https://go.sstools.co/u/{key} -> https://www.wikipedia.org`.

The short-URL prefix `https://go.sstools.co/u/` is a fixed constant (the
production namespace from the PRD), intentionally not derived from `BASE_URL`,
so the printed line matches the acceptance criteria regardless of the local
dev host.

## Gotchas

- `links.GenerateUniqueKey` takes the closure as `func(key string) (bool, error)`
  where the bool means "already taken" — the exists check returns the result of
  `SELECT EXISTS(...)` directly (true = taken), not negated.
- The dedup query must filter on `denied_reason = 0`; per the `links` schema a
  re-submitted denied URL should be re-evaluated, not silently reused, which is
  why the partial index `idx_links_user_destination` is scoped the same way.
