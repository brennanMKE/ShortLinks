## Description

Implement a simple health check endpoint that returns the service status and database connectivity. Used by monitoring tools and the systemd watchdog.

## Acceptance Criteria

- [ ] `GET /health` returns `200 OK` with JSON `{"status":"ok","db":"ok"}` when the DB is reachable
- [ ] Returns `503 Service Unavailable` with `{"status":"degraded","db":"error","error":"..."}` when the DB ping fails
- [ ] DB check uses `db.PingContext` with a 2-second timeout
- [ ] Response `Content-Type: application/json`
- [ ] No authentication required
- [ ] `go test ./internal/handlers/` covers healthy and degraded cases

## Notes

Dependencies: #0005, #0006.

## Implementation

No prior DB pool or HTTP server wiring existed (`internal/db` was a doc stub and
`cmd/shortlinks/main.go` only printed the version), so this issue lands the
foundational pieces alongside the health endpoint:

- **DB pool** (`internal/db/pool.go`): added `Connect(ctx, dsn) (*pgxpool.Pool, error)`
  using `github.com/jackc/pgx/v5/pgxpool`. It parses the DSN, builds the pool, and
  pings once at startup so a bad DSN or unreachable database fails fast rather than
  on the first request. The pool's own `Ping` is reused by the health handler.
- **Health handler** (`internal/handlers/health.go`): `HealthHandler` serves
  `GET /health`. It depends on a small `Pinger` interface (`Ping(ctx) error`),
  which `*pgxpool.Pool` satisfies, so it is unit-testable with a fake. The DB
  check runs under a 2-second timeout. Healthy → `200` with `{"status":"ok","db":"ok"}`;
  ping failure → `503` with `{"status":"degraded","db":"error","error":"..."}`.
  `Content-Type: application/json`; no authentication.
- **`serve` subcommand** (`cmd/shortlinks/main.go`): added subcommand routing so
  `shortlinks serve` loads config via `config.Load()`, connects the pool, mounts
  `GET /health` on an `http.ServeMux`, and listens on the configured `PORT`. The
  no-arg / `version` case still prints the version.

The JSON field names follow this issue's acceptance criteria (`db`, not
`database`) over the PRD's looser example.

## Gotchas

- `kill $SERVER_PID` only kills the `go run` parent, not the compiled child it
  spawns, so the listener can linger on 8080. Use `pkill -f 'shortlinks serve'`
  (or build a binary and run that) to be sure the port is freed.
- Sourcing `.env` in zsh trips on `EMAIL_FROM=ShortLinks <noreply@sstools.co>`
  (the `<...>` and spaces) with a parse error, but `DATABASE_URL`/`PORT` are still
  exported before the failing line, so the server starts fine. Export the needed
  vars inline if a clean source is required.
