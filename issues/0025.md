## Description

Implement the `internal/audit/` package that writes entries to the `audit_log` table. Every significant action in the system — authentication events, link lifecycle, admin actions — must write an audit entry.

## Acceptance Criteria

- [x] `internal/audit/audit.go` exposes a write API over `Entry{ActorID *int64, UserID *int64, Action string, TargetType string, TargetID *int64, Metadata any, IP string}`. The pool is held by the `Logger` rather than passed per call (`audit.New(pool)` then `Write/WriteTx/Record`), which reads cleaner than threading `db` through every call site; the `Entry` fields match the AC.
- [x] `Metadata` is serialized to JSONB; `nil` metadata stores NULL (also nil actor/user/target and empty IP store NULL).
- [x] All actions in the PRD action catalogue are defined as typed constants in `internal/audit/actions.go` (plus target-type constants).
- [x] All call sites (auth services, links/settings/url_filters/credentials handlers) write through the `*audit.Logger` — no direct SQL inserts into `audit_log` from other packages.
- [x] Audit writes never fail the parent request: request handlers use `Record` (log-and-continue); auth ceremonies use `WriteTx` so the row commits/rolls back atomically with the action it records.
- [x] `go test ./internal/audit/` covers: write with full metadata + JSONB round-trip, write with nil actor/target/metadata (NULL), and the admin-on-other-user shape.

## Implementation

Built `internal/audit` (package `audit`): an append-only writer over the shared
pgx pool.

- `actions.go` — exported action constants for the whole PRD catalogue
  (e.g. `ActionLinkCreated = "link.created"`) plus target-type constants
  (`TargetLink`, `TargetUser`, `TargetCredential`, `TargetSettings`,
  `TargetURLFilter`).
- `audit.go` — `Logger` (`New(pool)`) and `Entry{ActorID, UserID, Action,
  TargetType, TargetID, Metadata, IP}`. Nullable pointer fields and an empty IP
  insert SQL NULL; `Metadata` is JSON-marshalled into the JSONB column (nil →
  NULL). Three write paths:
  - `Write(ctx, Entry) error` — through the pool, returns the error (tests use it).
  - `WriteTx(ctx, tx, Entry) error` — through an already-open tx so the audit row
    commits/rolls back atomically with the action.
  - `Record(ctx, Entry)` — fire-and-forget: logs a swallowed error at WARN.

**Audit-failure policy (the choice this issue asked us to document):** handlers
whose action has already committed by the time we log use `Record`
(log-and-continue — a failed audit insert must never surface to the user). Auth
ceremonies (registration/login/recovery finish) already own a transaction for
account/credential/session creation, so their audit rows are written *in that
same transaction* via `WriteTx` — a committed action can never lose its audit
row, and a rolled-back action never leaves a stray one. Documented at the top of
`audit.go`.

The `*audit.Logger` is injected into `RegistrationService`, `LoginService`,
`RecoveryService`, `LinksHandler`, `SettingsHandler`, `URLFiltersHandler`, and
`CredentialsHandler` (constructors extended; a nil auditor disables writes for
unit tests) and constructed once in `cmd/shortlinks/main.go`. The actor IP is
extracted at the HTTP boundary with the existing `clientIP`
(X-Forwarded-For → RemoteAddr) helper and threaded into the service methods
(start/logout/finish take an `ip` param; finish methods already had the request).
`Store.DeleteSession` now returns the deleted session's `user_id` so
`account.logout` can be attributed.

Every seam was filled with the correct actor/user semantics and the exact PRD
metadata shape; the `TODO(#0025)`/`SEAM #0025` comments were removed.
`account.deactivated`/`account.reactivated` constants exist for #0028 but have no
call site yet (admin user management is not built). The #0026 SSE seam in
`LinksHandler.Create` and the #0026 references elsewhere were left untouched.

## Gotchas

- **Audit-failure policy is two-sided** (see Implementation): `Record` for
  already-committed handler actions, `WriteTx` for the auth ceremonies that own a
  transaction. An in-tx audit insert failure aborts the surrounding tx
  (poisoning the pgx connection), so `WriteTx` returns its error and is treated
  like any other ceremony step — this is why it is in-band rather than
  fire-and-forget there.
- **Typed-nil interface trap:** passing a nil `*cache.RuleCache` into the
  `ruleProvider` interface yields a non-nil interface wrapping a nil pointer, so
  `h.rules != nil` is true and `Rules` panics. Test mux helpers assign the typed
  pointer to the interface var only when non-nil (mirroring how `main.go`/the
  existing `linksMux` pass an untyped nil).
- `account.deactivated`/`account.reactivated` await #0028 (admin user
  management) — constants are defined and the audit API is trivially usable for
  the admin-on-other-user shape (proved by `TestWrite_PartialActorWithUser`), but
  there is no seam to fill yet.
