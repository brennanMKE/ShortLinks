# 0025 — Audit log — append-only write path for all system actions

|---|---|

## Description

Implement the `internal/audit/` package that writes entries to the `audit_log` table. Every significant action in the system — authentication events, link lifecycle, admin actions — must write an audit entry.

## Acceptance Criteria

- [ ] `internal/audit/audit.go` exposes `Write(ctx, db, Entry) error` where `Entry` has fields: `ActorID *int64`, `UserID *int64`, `Action string`, `TargetType string`, `TargetID *int64`, `Metadata any`, `IPAddress string`
- [ ] `Metadata` is serialized to JSONB; `nil` metadata stores NULL
- [ ] All actions in the PRD action catalogue are defined as typed constants in `internal/audit/actions.go`
- [ ] All call sites throughout the codebase (auth, links, filters, admin handlers) use `audit.Write` — no direct SQL inserts into `audit_log` from other packages
- [ ] Audit writes are fire-and-forget from the caller's perspective: errors are logged but do not fail the parent request
- [ ] `go test ./internal/audit/` covers: write with full metadata, write with nil actor, metadata serialized correctly
