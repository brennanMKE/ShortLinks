## Description

`docs/` currently holds only `email_setup.md`. Build out comprehensive developer/operator documentation so a newcomer (or a future Claude session) can understand and work on every part of Short Links. Each subsystem gets its own `docs/<topic>.md`, tracked by the sub-issues below, plus a `docs/README.md` index linking them all and a pointer from the top-level `README.md`.

## Motivation

The project spans a Go HTTP service, PostgreSQL, passkey/WebAuthn auth, a Svelte 5 SPA, click analytics, UTM tooling, URL filtering, an audit log, SSE, and caching — currently undocumented beyond `README.md`, `DEPLOYMENT.md`, and `docs/email_setup.md`. Per-topic docs make onboarding, maintenance, and AI-assisted work far easier and reduce the kind of re-derivation that caused recent friction.

## Conventions for all docs

- Live in `docs/`, markdown, written for a developer/operator audience.
- Cite real file paths (e.g. `internal/clicks/stats.go`) and keep accurate to the current code.
- Linked from `docs/README.md` (a new index) and the top-level `README.md`.
