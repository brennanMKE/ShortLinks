## Description

Implement UTM analytics: aggregate click counts broken down by utm_source, utm_medium, and utm_campaign. Surfaced via the link detail API endpoint.

## Acceptance Criteria

- [ ] `GET /api/links/{key}` response includes a `utm_stats` field with aggregated counts:
  ```json
  {
    "utm_stats": {
      "by_source": [{"value": "email", "count": 42}, ...],
      "by_medium": [{"value": "newsletter", "count": 30}, ...],
      "by_campaign": [{"value": "launch", "count": 42}, ...]
    }
  }
  ```
- [ ] NULL utm values are grouped as `"(none)"` in the aggregation
- [ ] Results ordered by count descending, limited to top 20 per dimension
- [ ] Total click count included in response: `"click_count": <int>`
- [ ] Queries run against the `clicks` table (no separate aggregate table needed for Phase 4)
- [ ] `go test ./internal/handlers/` covers: link with UTM clicks returns correct breakdown, link with no clicks returns empty arrays
