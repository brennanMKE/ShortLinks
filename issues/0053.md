# 0053 — Expires field on the create-link form is wider than the other text fields

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-22 |

## Description

On the Dashboard "Create a short link" form, the **Expires (optional)** input is wider than the other text inputs (Destination URL, Title (optional), Custom alias (optional)). It runs to the panel's right edge while the others stop short, so the fields no longer line up. Expires should be the same width as the other text fields.

## Steps to reproduce

1. Sign in at https://go.sstools.co/ and stay on the Dashboard.
2. In the "Create a short link" panel, compare the **Expires (optional)** field to Destination URL / Title / Custom alias.
3. Observe the Expires box is wider (flush to the panel edge) than the others.

## Expected behavior

The Expires input is the same width as the other text inputs in the form.

## Actual behavior

The Expires input is wider than the other inputs, breaking the form's alignment.

## Notes

Likely the Expires control (a `datetime-local` input) isn't constrained to the same width / `box-sizing` as the other fields. Shared input styling lives in `web/src/app.css` (global form-control rules); the create form is in `web/src/views/Dashboard.svelte`. Check whether the Expires input sits in the same `.field` wrapper and gets `width: 100%` like the others, or whether the native datetime input's intrinsic sizing is overriding it. Surfaced on mobile Safari in dark mode, but the discrepancy is layout, not theme-specific.
