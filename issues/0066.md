## Description

Write `docs/auth.md` documenting the session-based authentication and authorization layer (the passkey *ceremonies* themselves are covered separately in #0067).

## Scope

- Session lifecycle: how sessions are created, stored (`sessions` table), carried (cookie + `SESSION_SECRET`), validated, and expired.
- The session-guard middleware and how protected vs public routes are distinguished.
- The registration gate (`registrations_enabled` setting) and what it controls.
- Admin authorization: how `is_admin` gates the Admin tab/endpoints.
- Any rate limiting / logging middleware relevant to auth.
