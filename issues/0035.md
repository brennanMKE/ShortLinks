## Description

Implement the Link Detail view (`web/src/views/LinkDetail.svelte`). Shows full stats for a single link including click count, UTM breakdown, and controls to copy the short URL or deactivate the link.

## Acceptance Criteria

- [ ] Loaded when `currentView === "link-detail"`; reads selected link from `selectedLink` store
- [ ] Calls `GET /api/links/{key}` to load full detail including `utm_stats`
- [ ] Displays: short URL (with copy-to-clipboard button), destination URL, title, created_at, expires_at, active status, total click count
- [ ] UTM breakdown section shows top sources, mediums, and campaigns in sorted lists (by count descending); skipped if all counts are zero
- [ ] "Deactivate" button calls `DELETE /api/links/{key}`; on success updates link in `links` store and navigates back to `"dashboard"`
- [ ] "Back" button navigates to `"dashboard"` without a reload
- [ ] Denied links show the denial reason label prominently
- [ ] Component is shown when `currentView === "link-detail"`

## Notes

Dependencies: #0022 (link detail API), #0030 (UTM analytics in response), #0031 (stores, navigation).

The AC mentions a `selectedLink` store, but the actual store (lib/stores.ts) is `selectedLinkKey` (a `string | null` key, not a full link object), because there is no URL router and the detail view must load the link itself. The implementation follows the real store contract.

## Implementation

`web/src/views/LinkDetail.svelte` (previously a one-line placeholder) was built out with Svelte 5 runes, matching the style/error-handling of `Dashboard.svelte` and `Login.svelte`:

- On mount it reads `$selectedLinkKey`; a null key short-circuits to the "not found" state (no perpetual spinner). Otherwise it calls `getLink(key)` (`GET /api/links/{key}`).
- States: loading, `notFound` (404 → "Link not found" with a Back action), generic `loadError` (with Retry), and the loaded detail. A 401 redirects to login.
- Displays the short URL `https://go.sstools.co/u/{key}` (via `shortUrl`) with a copy-to-clipboard button, the destination URL, title (or "Untitled link"), a single-string status badge (Active / Inactive / `Denied: <reason>`), created + expiry dates ("Never" when no expiry), and total click count.
- UTM breakdown section renders `by_source` / `by_medium` / `by_campaign` as labeled tables of value → count, sorted by count desc (value asc tiebreaker, matching the server). The `(none)` bucket is rendered with a muted italic label; the whole section collapses to a single "No click data yet" message when stats are empty/absent (AC: skip when all counts are zero).
- "Deactivate link" calls `deactivateLink(key)` (`DELETE /api/links/{key}`), updates the displayed status and the shared `links` store row in place, and is hidden once the link is inactive/denied.
- "Dashboard" (back) clears `selectedLinkKey` and sets `currentView = "dashboard"` — a store write, no reload.

Pure logic was factored into `web/src/lib/linkDetail.ts` (no DOM/network) and unit-tested: `isNoneBucket`, `sortBuckets`, `isEmptyStats`, `utmDimensions`, `statusLabel` (reusing `linkStatus` + `deniedReasonLabel` from `lib/links.ts`), and `formatDate`.

Confirmed against the backend contract: `GET /api/links/{key}` returns `linkView` fields plus an optional `utm_stats` object whose `by_source`/`by_medium`/`by_campaign` are arrays of `{value, count}` (`internal/clicks/stats.go` `UTMStats`/`Bucket`); the none bucket label is `"(none)"`; a missing/other-user link is 404 `{error:"link not found"}`. All match `web/src/lib/types.ts` (`LinkDetail`/`ClickStats`/`UTMBucket`) — no type changes were needed.

## Gotchas

- The clicks store folds NULL and empty UTM values into the literal `"(none)"` sentinel (`internal/clicks/stats.go` `NoneBucket`), so the client matches on that exact string (and, defensively, on empty/whitespace) rather than on `null`.
- `utm_stats` is optional in the response (`omitempty` when no analytics store is wired), so every helper accepts `undefined`/`null` and the view gates the whole section on `isEmptyStats`.
- The issue text says `selectedLink`; the real store is `selectedLinkKey` — see Notes.
