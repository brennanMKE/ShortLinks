# 0054 — Replace the hardcoded "go.sstools.co" branding in the web UI with the product name "Short Links"

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-22 |

## Description

The web UI displays the deployment domain `go.sstools.co` as the app's name/title — the header brand on Dashboard/Account/Admin and the login/verify page headings. ShortLinks is a self-hostable project that anyone can deploy under their own domain, so the UI should show the product name **"Short Links"** instead of one specific instance's hostname.

## Motivation

`go.sstools.co` is *this* deployment's domain, not the product. Baking it into the shipped UI means a fresh deployment under a different domain would still read "go.sstools.co" (or surface its own hostname where a product name belongs). Using the static brand "Short Links" makes the UI correct for every deployment.

## Proposed approach

- Replace the displayed app title/brand with the literal "Short Links" everywhere the domain is currently shown *as the name*: the header `app-title` (`App`/`Dashboard`/`Account`/`Admin`) and the `Login` / `RegisterVerify` / `RecoverVerify` headings.
- Leave functional URLs untouched — the domain still legitimately appears in generated short URLs and the WebAuthn RP origin; only the **branding/name** usage changes.
- Find where the title string originates (likely `BASE_URL` or the RP ID rendered as a heading) and swap it for a static product name; consider a single shared constant so the brand is defined once.

## Acceptance criteria

- [ ] The header brand and auth-page headings read "Short Links", not the deployment domain
- [ ] No functional URLs (generated short links, RP origin/ID) are changed
- [ ] `cd web && npm run build` and `npm run check` clean
