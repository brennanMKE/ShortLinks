## Description

Implement the paginated audit log endpoint for admin review. Supports optional filtering by user ID.

## Acceptance Criteria

- [ ] `GET /admin/audit`: returns paginated audit log in reverse chronological order; supports `?page=` and `?per_page=` (default 50); requires admin auth
- [ ] Optional `?user_id=<int>` query parameter filters to a specific user's actions
- [ ] Each item in response includes: `{id, actor_id, user_id, action, target_type, target_id, metadata, ip_address, created_at}`
- [ ] `metadata` returned as raw JSON object (not a string)
- [ ] Returns `200` with `{"audit_log": [...], "total": <int>, "page": <int>, "per_page": <int>}`
- [ ] Requires admin auth (`RequireAdmin` middleware from #0017)
- [ ] `go test ./internal/handlers/` covers: full log returned, filtered by user_id, pagination

## Notes

Dependencies: #0013 (audit_log table with indexes), #0017 (admin auth middleware), #0025 (audit write path).

## Implementation

Added the read counterpart to the #0025 write path so the column semantics
(`actor_id` = who acted, `user_id` = whose resource was affected) stay defined
beside the writer:

- **`internal/audit/read.go`** — `audit.Reader.ListAuditLog(ctx, userID *int64, limit, offset)`
  returns a page of rows plus the total count. Rows are ordered
  `created_at DESC, id DESC` (the `id` tiebreak keeps same-timestamp rows
  stable). When `userID` is non-nil the query filters on `user_id` (the
  `(user_id, created_at DESC)` index); otherwise it scans the whole log (the
  `(created_at DESC)` index). `ip_address` is read via `host()` to the bare
  address; `metadata` is read as raw JSONB bytes (`json.RawMessage`) so it
  reaches the client as an object, and a SQL NULL stays nil.
- **`internal/handlers/audit.go`** — `AdminAuditHandler.List` (GET /admin/audit).
  Parses `?page=` and `?per_page=` (default 50, capped at 200; non-positive-int
  → 400) and an optional `?user_id=` (non-positive-int → 400). Nullable columns
  are pointer fields so they serialize as JSON `null`, never `0`/`""`. Response
  shape per the AC: `{"audit_log":[...],"total":N,"page":P,"per_page":PP}`.
- **`cmd/shortlinks/main.go`** — constructs `audit.NewReader(pool)` and wires
  `GET /admin/audit` behind `requireAdmin` (RequireSession → RequireAdmin).

## Gotchas

- `metadata` must be carried as `json.RawMessage` end-to-end (DB raw bytes →
  `audit.Record.Metadata` → `auditRecordView.Metadata`). Scanning into a `string`
  or marshalling a Go map would either re-quote the JSON or reorder keys; passing
  the raw JSONB bytes through is what keeps it a faithful JSON object.
- `ip_address` is `INET`; selecting it directly yields `203.0.113.7/32`. Reading
  it via `host(ip_address)` returns the bare address, matching how the #0025
  write-path tests assert it.
- The `id`-DESC tiebreak on the `ORDER BY` matters: without it, two rows sharing
  a `created_at` could swap between pages and break the pagination assertions.
