## Description

Write `docs/architecture.md` — the high-level map of how Short Links fits together and how a request flows through it, so a reader can orient before diving into subsystem docs.

## Scope

- Components: `cmd/shortlinks` (`serve` / `seed`), the `internal/*` packages, the Svelte SPA embedded via `web/embed.go`, PostgreSQL, the Apache reverse proxy, and the systemd service.
- Request lifecycles: (a) a short-link redirect `GET /u/{key}` through cache → DB → click recording; (b) an API request through middleware → handler → store; (c) SPA serving via the catch-all that returns `index.html`.
- Tech stack and key dependencies (pgx, golang-migrate, go-webauthn, Ristretto, Svelte 5 / Vite).
- A short "where to look" table mapping subsystems → packages → the per-topic doc.
