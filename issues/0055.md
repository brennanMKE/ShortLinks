# 0055 — Add a GitHub repository link to the SPA footer

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-22 |

## Description

Add a footer to the web UI with a link to the project's GitHub repository (`https://github.com/brennanMKE/ShortLinks`) so visitors can find the source, fork it, and deploy their own Short Links instance. The SPA currently has no footer or source link.

## Motivation

Short Links is an open, self-hostable project (see [#0054](0054.md)). Surfacing the repo link in the UI invites others to fork and self-deploy and credits the source. A small, unobtrusive footer is the conventional place for it.

## Proposed approach

- Add a footer to the app shell, shown across the main views and the login/verify pages, containing a link to `https://github.com/brennanMKE/ShortLinks` (open in a new tab, `target="_blank"` + `rel="noopener noreferrer"`).
- Style it with the #0052 design tokens (muted text, small font); keep it unobtrusive, responsive, and correct in both light and dark mode.
- Wording can pair with the "Short Links" brand from #0054 — e.g. "Short Links — open source on GitHub", or a GitHub glyph + "Source".

## Acceptance criteria

- [ ] A footer with a working link to `https://github.com/brennanMKE/ShortLinks` (new tab, `rel="noopener noreferrer"`) appears in the UI
- [ ] Styled with tokens; legible in light and dark mode; unobtrusive and responsive
- [ ] `cd web && npm run build` and `npm run check` clean

## Relation

- Pairs with: [#0054](0054.md) ("Short Links" branding — both make the UI fit a self-deployable, open project)
