## Description

Implement the Admin view (`web/src/views/Admin.svelte`). Only visible when `currentUser.is_admin === true`. Contains four sections: server settings, URL filter rules, user list, and audit log.

## Acceptance Criteria

- [ ] Loaded when `currentView === "admin"`; hidden from navigation when `currentUser.is_admin === false`
- [ ] **Settings section**: loads `GET /admin/settings`; shows `registrations_enabled` as a toggle; on change calls `PATCH /admin/settings`; shows success/error feedback
- [ ] **URL filter rules section**: loads `GET /admin/url-filters`; lists rules with pattern, reason code label, active status; "Add rule" form: pattern input, reason code dropdown, description; inline edit and delete per rule; `POST /admin/url-filters/test` — URL input, shows matched rule or "no match"
- [ ] **User list section**: loads `GET /admin/users`; shows email, is_admin badge, active/inactive status, last_login_at; "Deactivate" button opens a modal: reason dropdown (6 options), optional note (required for "other"); calls `POST /admin/users/{id}/deactivate`; "Reactivate" button for inactive users calls `POST /admin/users/{id}/reactivate`
- [ ] **Audit log section**: loads `GET /admin/audit`; paginated table with actor, action, target, created_at; `?user_id=` filter input to narrow to a specific user
- [ ] All four sections present as tabs or collapsible panels within a single view
- [ ] Component is shown when `currentView === "admin"`

## Notes

Dependencies: #0021 (settings API), #0024 (filter rules API), #0028 (user management API), #0029 (audit log API), #0031 (stores).
