## Implementation

`web/src/views/Admin.svelte` (previously a placeholder) is now the full admin
view, gated on `currentUser?.is_admin` â€” a non-admin who reaches it sees an
"access denied" notice instead of any admin data. A sub-tab bar switches between
four stacked panels; all four load on mount.

- **Settings** â€” `GET /admin/settings`; the `registrations_enabled` value is
  shown as an On/Off toggle (`role="switch"`). Toggling PATCHes
  `/admin/settings` with `"true"`/`"false"`, applies the authoritative settings
  list the PATCH returns, and shows a success notice (or an error).
- **URL filters** â€” `GET /admin/url-filters` lists each rule (pattern, reason
  label, description, active). An "Add rule" form (pattern + the six-code reason
  dropdown + description) POSTs and surfaces the server's 400 regex/validation
  message inline. Each row supports inline edit (PATCH) and delete (DELETE,
  behind a `confirm`). A test tool POSTs `/admin/url-filters/test` and renders
  the matched rule + reason label or a "would be allowed" message. Every
  mutation refreshes the list.
- **Users** â€” `GET /admin/users` lists email, admin flag, active/inactive
  status badge, and last login. Deactivate (offered only for active non-admins
  that are not the acting admin) opens a modal with the six-value reason
  dropdown and a note field whose label flips to "(required)" for `other`;
  client-side validation mirrors the server (`other` â‡’ note required). Inactive
  users get a Reactivate action with an optional note prompt. Status updates in
  place from the returned user.
- **Audit log** â€” `GET /admin/audit?page=&per_page=50&user_id=`; a
  newest-first table (when, action, actor, target, metadata, ip) with a numeric
  per-user filter (Apply/Clear) and Previous/Next paging showing the
  "Nâ€“M of T Â· page P of Q" range.

All non-trivial pure logic was factored into `web/src/lib/admin.ts` and
unit-tested in `web/src/lib/admin.test.ts`: the reason-codeâ†’label map and the
six `REASON_OPTIONS`, the deactivation reason values + labels, the
