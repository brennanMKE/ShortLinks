## Description

Implement the `/api/me` endpoint that returns the authenticated user's profile. Used by the Svelte SPA on load to determine whether to show the Admin tab.

## Acceptance Criteria

- [x] `GET /api/me`: requires valid session (uses `RequireSession` middleware)
- [x] Returns `200 {"id": <int>, "email": "<string>", "is_admin": <bool>}`
- [x] Returns `401` if not authenticated
- [x] `go test ./internal/handlers/` covers: authenticated returns profile with is_admin field, unauthenticated returns 401

## Notes

Dependencies: #0017 (auth middleware).
PRD reference: Key HTTP Routes table — `GET /api/me`.

## Implementation

Added a stateless `MeHandler` in `internal/handlers/me.go` exposing
`GET /api/me`. The endpoint reads the authenticated principal straight off the
request context — `middleware.UserFromContext` returns `*AuthUser{ID, Email,
IsAdmin}`, which is exactly the profile shape — so there is **no DB round-trip**
and no new `auth.Store` query was needed. The handler responds `200` with
`{"id", "email", "is_admin"}` (snake_case `is_admin`, matching the PRD baseline
and the rest of the API). It also re-reads the context user and returns `401` if
absent, so it never panics if mounted without the guard.

The route is wired in `cmd/shortlinks/main.go` behind `requireSession`
(`mux.Handle("GET /api/me", requireSession(http.HandlerFunc(meH.Me)))`),
alongside the other `/api/links` routes. RequireSession alone produces the `401`
for an absent/invalid session, so the AC's unauthenticated case is enforced by
the middleware, not the handler.

## Gotchas

- The AC referenced `RequireAuth`; the actual middleware in this codebase is
  `middleware.RequireSession` (with `RequireAdmin` layered for admin routes).
  Used `RequireSession` since `/api/me` is for any authenticated user.
- The existing `seedUser` helper hardcodes `is_admin = FALSE`, so a dedicated
  `seedAdminUser` helper was added in `me_test.go` to cover the `is_admin:true`
  case.
