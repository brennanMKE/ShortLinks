# 0050 — Make the SPA responsive for iPhone (mobile) screen sizes

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-18 |

## Description

Adapt the management SPA to iPhone-class viewport widths (~375–430px) so links can be created, viewed, and managed comfortably on a phone. The `<meta name="viewport">` is already present (`web/index.html:5`), but layouts use fixed `max-width` cards and the list/table views (Dashboard link list, Admin, LinkDetail) are not audited for narrow screens.

## Motivation

The app is reached at `go.sstools.co` from any device; short-link management is a natural thing to do from a phone. Today's fixed-width cards and wide tables overflow or get cramped on mobile.

## Proposed approach

- Audit each view (`Login`, `Dashboard`, `LinkDetail`, `Account`, `Admin`, `RegisterVerify`, `RecoverVerify`) at iPhone widths in browser device emulation.
- Replace fixed pixel/rem widths with fluid constraints (`max-width` + `width: 100%`, sensible padding); add breakpoints where layout must reflow.
- Make tabular/list data (link lists, audit log, per-UTM stats) reflow or scroll deliberately rather than overflow the viewport.
- Ensure tap targets (buttons, links) meet a usable minimum size; check the passkey ceremony flows render on mobile Safari.
- Verify any charts from [#0049](0049.md) scale down without breaking.

## Acceptance criteria

- [ ] All views usable and non-overflowing at 375px and 430px widths
- [ ] Link create/list, LinkDetail, Account, Admin reflow cleanly; no horizontal scroll except where intentional
- [ ] Tap targets are comfortably sized on mobile
- [ ] Login + passkey ceremonies work on mobile Safari
- [ ] `cd web && npm run build` and `npm run check` clean
