## Description

Create the Apache 2 virtual host configuration file that proxies all traffic for `go.sstools.co` to the Go service on `localhost:8080`. The SSE endpoint requires special treatment to disable response buffering.

## Acceptance Criteria

- [ ] Config file created at `deploy/apache/go.sstools.co.conf`
- [ ] Port 443 virtual host with `ProxyPreserveHost On`
- [ ] `/api/events` proxied with `flushpackets=on` (listed before the wildcard `ProxyPass`)
- [ ] All other traffic proxied to `http://127.0.0.1:8080/`
- [ ] SSL certificate paths reference Let's Encrypt paths for `go.sstools.co`
- [ ] `deploy/apache/README.md` documents:
  - Required Apache modules (`a2enmod proxy proxy_http ssl`)
  - How to install: `sudo cp go.sstools.co.conf /etc/apache2/sites-available/ && sudo a2ensite go.sstools.co && sudo systemctl reload apache2`
  - How to obtain the Let's Encrypt certificate (`certbot --apache -d go.sstools.co`)

## Notes

PRD reference: Apache Configuration section.
The `flushpackets=on` directive on the SSE route must appear before the wildcard `ProxyPass /` line, otherwise Apache matches the wildcard first and buffering is not disabled.
