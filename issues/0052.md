## Description

The management SPA is functional but visually minimal — plain cards, default-ish typography, ad-hoc spacing, and little visual hierarchy or polish. Building on the responsive layout work (#0050) and the light/dark theming work (#0051), give the SPA a deliberate, cohesive design: a defined visual language (type scale, spacing rhythm, color, component styling, interactive states) that makes the app feel considered and on-brand rather than a bare scaffold.

## Motivation

This is the public-facing management UI at `go.sstools.co`, reached from any device. A more refined design improves usability, trust, and perceived product quality. #0050 establishes the structural (responsive) groundwork and #0051 establishes the theming token groundwork; this issue is the layer on top — hierarchy, rhythm, component consistency, states, and overall polish — so the three together turn a minimal scaffold into a finished-feeling app.

## Proposed approach

Direction is the **Portal-Style Design Guide** in Reference below ([0052/svelte-portal-design-guide.md](0052/svelte-portal-design-guide.md)) — a polished-but-simple, portal-era aesthetic (GitHub Primer / early-Facebook / Yahoo lineage): boundaries over decoration, one accent color, an 8px spacing grid, a tight type scale.

- Add the SPA's first global stylesheet (`web/src/app.css`) holding the guide's design tokens (neutral palette, single accent, 8px spacing scale, type scale, 4px radius, 1px borders) on `:root`; import it once from `main.ts`.
- Favor **boundaries over decoration**: 1px borders and subtle background shifts instead of shadows / gradients / animation; one accent for everything interactive, neutrals for the rest.
- Build reusable primitives — `web/src/lib/Panel.svelte` (bounded box) and `web/src/lib/Button.svelte` (default / primary / subtle / danger) — plus global form-control and dense-bordered-table styling in `app.css`.
- Adopt a centered `app-shell` column layout; migrate each view (`App`, `Login`, `Dashboard`, `LinkDetail`, `Account`, `Admin`, `RegisterVerify`, `RecoverVerify`) onto the tokens, `Panel`, `Button`, and global controls — removing per-component hardcoded colors / spacing.
- Structure tokens as CSS custom properties so a later `prefers-color-scheme: dark` override (#0051) only restates the neutral/accent values — no per-component change.
- Keep it lightweight — Svelte 5, no UI framework; honor the embed-in-Go-binary constraint and the existing Vite build.

## Gotchas

- Svelte 5 has no `on:click` forwarding; `Button` takes `onclick` via `$props()` and spreads `{...rest}` — follow that pattern, not the guide's Svelte 4 snippet.
- Global `label` is bold/muted and inputs are `width:100%`; inside table cells these need local overrides (see `Admin.svelte`) — expected.
- A few status tints remain literal hex (`Button.svelte` danger-hover `#fbeaea`, Dashboard denied-box `#fdecea`, app.css badge tints); each needs a one-line dark restate when #0051 lands.
- Non-blocking follow-ups noted by the reviewer: dropped disabled-revoke `title` tooltip in `Account.svelte`, lost `.num` right-align in the LinkDetail UTM table, and several explanatory comments removed during migration.
