## Description

Implement the Account view (`web/src/views/Account.svelte`) for managing registered passkeys: viewing all credentials, renaming them, and revoking them (with a guard against revoking the last one).

## Acceptance Criteria

- [ ] Loaded when `currentView === "account"`
- [ ] On mount, calls `GET /account/credentials` to list passkeys
- [ ] Each credential row shows: `device_name`, `created_at`, `last_used_at`; has inline rename (editable field + save) and revoke (button + confirmation) actions
- [ ] Rename: calls `PATCH /account/credentials/{id}` with `{device_name}`; updates list in place on success
- [ ] Revoke: calls `DELETE /account/credentials/{id}`; if server returns `409` (last credential), shows error "You cannot revoke your only passkey. Add another passkey first."
- [ ] On successful revoke, removes credential from list
- [ ] Component is shown when `currentView === "account"`

## Notes

Dependencies: #0019 (credential management API), #0031 (stores, navigation).
