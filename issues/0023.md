# 0023 — URL deduplication — per-user duplicate detection on link creation

| | |
|---|---|
| **Status** | open |
| **Module** | links |
| **Platform** | All |
| **First seen** | 2026-05-25 |

## Description

Implement per-user URL deduplication in the link creation path. When a user submits a destination URL they have previously shortened (and not had denied), return the existing link rather than inserting a duplicate. Inactive links are reactivated.

## Acceptance Criteria

- [ ] Dedup query runs after filter check and only when no custom key was supplied:
  ```sql
  SELECT * FROM links WHERE user_id = $1 AND destination_url = $2 AND denied_reason = 0 LIMIT 1
  ```
- [ ] Active link found → return existing link with `"duplicate": true`; no insert; no SSE event; writes `link.created` audit entry with `{duplicate: true}` metadata
- [ ] Inactive link found (`active = false`, `denied_reason = 0`) → set `active = true`; return reactivated link with `"duplicate": true`; broadcast `link.created` SSE event; writes `link.reactivated` audit entry
- [ ] No match → insert new link; return with `"duplicate": false`; broadcast `link.created` SSE event
- [ ] Custom aliases (`key` provided in request body) bypass dedup entirely — insert is attempted regardless
- [ ] `go test ./internal/links/` covers: active duplicate, inactive reactivation, no match, custom alias bypasses dedup
