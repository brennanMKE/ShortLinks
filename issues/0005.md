## Description

Implement `internal/config/` to load all environment variables at startup, validate required fields, and expose a typed `Config` struct injected into the rest of the application. In development the values come from `.env` via `godotenv`; in production from the process environment set by systemd.

## Acceptance Criteria

- [ ] `Config` struct with fields for all variables in `.env.example`: `Port`, `BaseURL`, `DatabaseURL`, `WebAuthnRPID`, `WebAuthnRPOrigin`, `SessionSecret`, `SESSmtpHost`, `SESSmtpPort`, `SESSmtpUsername`, `SESSmtpPassword`, `EmailFrom`, `CacheMaxCost`, `CacheTTLSeconds`, `AdminEmail`
- [ ] `Load() (*Config, error)` loads `.env` if the file exists (dev), then reads `os.Getenv` for all fields
- [ ] Returns an error listing every missing required field (not just the first one)
- [ ] Required fields: `DatabaseURL`, `WebAuthnRPID`, `WebAuthnRPOrigin`, `SessionSecret`, `AdminEmail`
- [ ] Optional fields use sensible defaults: `Port=8080`, `CacheMaxCost=10000`, `CacheTTLSeconds=300`
- [ ] `go test ./internal/config/` passes, covering missing-required-field and defaults cases

## Implementation

Built `internal/config` on top of the existing `doc.go` package stub.

- `Config` struct with a typed field for every variable in `.env.example`:
  `Port`, `BaseURL`, `DatabaseURL`, `WebAuthnRPID`, `WebAuthnRPOrigin`,
  `SessionSecret`, `SESSmtpHost`, `SESSmtpPort`, `SESSmtpUsername`,
  `SESSmtpPassword`, `EmailFrom`, `CacheMaxCost`, `CacheTTLSeconds`,
  `AdminEmail`. Ints: `Port`, `SESSmtpPort`, `CacheMaxCost`,
  `CacheTTLSeconds`; all others string.
- `Load() (*Config, error)` loads `.env` via `godotenv` when the file exists
  (a missing `.env` is intentionally *not* an error, since systemd sets the
  vars in production), then reads `os.Getenv` for each field.
- Defaults per PRD/`.env.example`: `Port=8080`, `SESSmtpPort=587`,
  `CacheMaxCost=10000`, `CacheTTLSeconds=300`.
- Integer parsing via `strconv.Atoi`; a set-but-unparseable value is an error.
- Validation collects **every** missing required field and every bad integer
  into a single combined error rather than failing on the first.
  Required: `DATABASE_URL`, `WEBAUTHN_RP_ID`, `WEBAUTHN_RP_ORIGIN`,
  `SESSION_SECRET`, `ADMIN_EMAIL`.

## Fix

Added `internal/config/config.go` (loader) and
`internal/config/config_test.go` (table-driven tests). Added the
`github.com/joho/godotenv v1.5.1` dependency to `go.mod`/`go.sum`.

An unexported `loadFromFile(path string)` backs the exported `Load()`,
letting tests point at a non-existent path so they exercise only env vars
set via `t.Setenv` (no leakage, no reliance on a real `.env`).

## Gotchas

- `go mod tidy` removes an unimported dependency, so `go get godotenv` before
  writing the import temporarily dropped it; tidying *after* the import is in
  place re-adds it correctly.
- A missing `.env` must not be treated as a failure — production sets the
  variables directly via the systemd unit, with no file on disk.
