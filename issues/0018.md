## Description

Implement account recovery for users who have lost access to all their registered passkeys. The flow sends a recovery email with a time-limited link that lets the user register a new credential on their existing account.

## Acceptance Criteria

- [ ] `POST /auth/recover`: accepts `{email}`; looks up user (returns `200` even if email not found, to prevent enumeration); sends recovery email with single-use token (15-min TTL); stores token in `pending_registrations` (reuse same table, distinguish by a `purpose` field or separate token type); returns `200 {"message":"If that email is registered, a recovery link has been sent"}`
- [ ] `GET /auth/recover/verify?token=...`: validates token; returns WebAuthn `PublicKeyCredentialCreationOptions` for adding a new credential (same params as registration: `residentKey: "required"`, `userVerification: "required"`, `authenticatorAttachment` omitted); stores challenge in `webauthn_challenges`
- [ ] `POST /auth/recover/finish`: verifies challenge; inserts new `passkey_credentials` row linked to existing `users.id`; does NOT create a new user; does NOT remove old credentials; creates session; sets cookie
- [ ] Writes `account.recovery_started` audit entry (actor_id NULL) on start, `account.recovered` on finish
- [ ] Writes `credential.added` audit entry on finish
- [ ] `go test ./internal/handlers/` covers: email not found (no enumeration), valid recovery, expired token, invalid challenge
