## Description

Implement the Server-Sent Events broker (`internal/events/`) and the authenticated `/api/events` endpoint. The broker fans out `link.created` events to all connected dashboard clients for a given user.

## Acceptance Criteria

- [ ] `internal/events/broker.go` implements `Broker` with `Subscribe(userID int64) chan Event`, `Unsubscribe(userID int64, ch chan Event)`, and `Publish(userID int64, event Event)` — safe for concurrent use
- [ ] `Event` struct has `Name string` and `Payload []byte` (JSON-encoded link row)
- [ ] `GET /api/events`: requires valid session cookie; sets headers `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `X-Accel-Buffering: no`; calls `Subscribe(userID)` to get a channel; writes SSE frames on each received event; calls `Unsubscribe` and exits goroutine when `r.Context().Done()` fires
- [ ] SSE frame format: `event: <Name>\ndata: <Payload>\n\n`
- [ ] Broker is a singleton created in `main` and injected into the link handler (publisher) and SSE handler (subscriber)
- [ ] `POST /api/links` success path calls `broker.Publish(userID, Event{Name: "link.created", Payload: <link JSON>})`
- [ ] `go test ./internal/events/` covers: publish reaches subscriber, unsubscribe stops delivery, multiple subscribers same user
