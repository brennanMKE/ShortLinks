## Description

Implement the Server-Sent Events broker (`internal/events/`) and the authenticated `/api/events` endpoint. The broker fans out `link.created` events to all connected dashboard clients for a given user.

## Acceptance Criteria

- [x] `internal/events/broker.go` implements `Broker` with `Subscribe(userID int64) chan Event`, `Unsubscribe(userID int64, ch chan Event)`, and `Publish(userID int64, event Event)` — safe for concurrent use
- [x] `Event` struct has `Name string` and `Payload []byte` (JSON-encoded link row)
- [x] `GET /api/events`: requires valid session cookie; sets headers `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `X-Accel-Buffering: no`; calls `Subscribe(userID)` to get a channel; writes SSE frames on each received event; calls `Unsubscribe` and exits goroutine when `r.Context().Done()` fires
- [x] SSE frame format: `event: <Name>\ndata: <Payload>\n\n`
- [x] Broker is a singleton created in `main` and injected into the link handler (publisher) and SSE handler (subscriber)
- [x] `POST /api/links` success path calls `broker.Publish(userID, Event{Name: "link.created", Payload: <link JSON>})`
- [x] `go test ./internal/events/` covers: publish reaches subscriber, unsubscribe stops delivery, multiple subscribers same user

## Implementation

Built the in-memory broker in `internal/events/broker.go` to the PRD shape:
`Event{Name string; Payload []byte}` and `Broker{mu sync.RWMutex; subscribers
map[int64][]chan Event}`. `Subscribe` returns a NEW buffered channel (buffer 8)
so a brief reader stall does not drop an event; `Unsubscribe` removes the
channel from the user's slice and closes it (idempotent — a double or unknown
unsubscribe is a safe no-op, never a double-close); `Publish` holds a read lock
and uses a non-blocking `select { case ch <- event: default: }` per subscriber
so a full/slow subscriber is skipped rather than blocking the publisher or the
other subscribers. Empty user slots are deleted to keep the map tidy.

Added `internal/handlers/events.go` (`EventsHandler.Stream`) behind
`RequireSession`: reads the user from context, obtains `http.Flusher` (500 if
unsupported, before any 200), sets `Content-Type: text/event-stream`,
`Cache-Control: no-cache`, `Connection: keep-alive`, `X-Accel-Buffering: no`,
writes an initial `: connected` comment + flush so proxies release headers,
then loops `select`-ing on the broker channel (writing
`event: <name>\ndata: <payload>\n\n` + flush) and on `r.Context().Done()`
(deferred Unsubscribe + return). The handler depends on a small
`eventSubscriber` interface for testability.

Wired the publisher seam in `LinksHandler.Create`: added an optional
`eventPublisher` field/arg to `NewLinksHandler`; when the existing `broadcast`
boolean is true (insert or reactivation) the handler marshals the SAME
`linkView` it returns and calls
`broker.Publish(u.ID, events.Event{Name:"link.created", Payload: payload})`.
Active-duplicate (`broadcast == false`) and filter-denied (returns earlier)
creates never publish. The stale `_ = broadcast` placeholder and the
"Not implemented" #0026 seam comments were removed. `cmd/shortlinks/main.go`
constructs the broker singleton and injects it into both handlers, and mounts
`GET /api/events` behind `requireSession`.

## Gotchas

- `Publish` MUST be non-blocking. A plain `ch <- event` would let one stalled
  SSE reader (its buffer full) block the publisher's HTTP request and every
  other subscriber. The `select { … default }` drops for the stuck subscriber
  only; `TestPublishNonBlockingOnFullBuffer` guards this.
- The publisher field is an `eventPublisher` **interface**, so a nil broker must
  be passed as the interface value (it is in `serve()` only when intentionally
  disabled). A typed-nil `*events.Broker` would be a non-nil interface and the
  `h.broker != nil` guard would call `Publish` on a nil pointer.
- The SSE handler test runs through the real `RequireSession` + a seeded DB
  session rather than injecting a context user, because `middleware` exposes no
  exported context-injector (`withUser` is unexported) — matching every other
  handler test in the package.
