## Description

Implement a `Mailer` interface and AWS SES SMTP implementation used to send transactional emails: magic-link verification emails and account recovery emails.

## Acceptance Criteria

- [ ] `internal/auth/mailer.go` defines a `Mailer` interface with at minimum: `SendVerification(ctx, toEmail, token string) error` and `SendRecovery(ctx, toEmail, token string) error`
- [ ] `internal/auth/ses_mailer.go` implements `Mailer` using SMTP (host/port/username/password from `Config`)
- [ ] Verification email body includes the full magic link: `{BASE_URL}/auth/register/verify?token={token}`
- [ ] Recovery email body includes the full recovery link: `{BASE_URL}/auth/recover/verify?token={token}`
- [ ] Uses `net/smtp` with STARTTLS on port 587 (SES_SMTP_PORT from config)
- [ ] `From` header uses `EMAIL_FROM` from config
- [ ] A `NoOpMailer` stub is provided for testing (logs to stdout, does not send)
- [ ] `go test ./internal/auth/` covers mailer with `NoOpMailer` (no real SES call required)

## Notes

Dependencies: #0005 (config loader), #0013 (auth tables — registration flow calls mailer).
See `EMAIL_SETUP.md` for SES SMTP credential setup and DNS configuration.
