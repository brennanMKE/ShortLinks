## Description

Implement the Login view component (`web/src/views/Login.svelte`). Supports passkey autofill via conditional UI and manual email-first login. Also handles the registration flow entry point.

## Acceptance Criteria

- [ ] Email input uses `autocomplete="username webauthn"` for passkey autofill (conditional UI)
- [ ] On mount, calls `GET /auth/login/start` to pre-load a challenge (enables browser passkey autofill)
- [ ] "Sign in" button calls `GET /auth/login/start?email=<input>` then triggers `navigator.credentials.get()` with returned options, then `POST /auth/login/finish`; on success sets `currentUser` and navigates to `"dashboard"`
- [ ] "Register" link/button shows a registration sub-form: email input → `POST /auth/register/start` → displays "Check your email" message
- [ ] Error states displayed inline: deactivated account, rate limit exceeded, registration closed
- [ ] `go.sstools.co` branding shown
- [ ] Component is shown when `currentView === "login"`

## Implementation

Built the Login view plus a reusable WebAuthn browser helper.

- `web/src/lib/webauthn.ts` — the pure encoding/shaping glue:
  - `bufferToBase64url` / `base64urlToBytes`: unpadded base64url ↔
    `Uint8Array`, matching Go's `base64.RawURLEncoding`.
  - `toPublicKeyRequestOptions`: server request-options JSON →
    `PublicKeyCredentialRequestOptions`, decoding `challenge` and each
    `allowCredentials[].id`.
  - `serializeAssertion`: the `PublicKeyCredential` from
    `navigator.credentials.get()` → the exact JSON `POST /auth/login/finish`
    parses (`id`, `type`, `rawId`, `clientExtensionResults`,
    `response.{clientDataJSON,authenticatorData,signature,userHandle}`),
    omitting `userHandle` when absent/empty.
  - `conditionalMediationAvailable`: feature-detect passkey autofill.
- `web/src/views/Login.svelte` — Svelte 5 runes. Email input with
  `autocomplete="username webauthn"`; an on-mount background
  `mediation: 'conditional'` ceremony (passkey autofill) that is aborted on
  unmount; an explicit email-first "Sign in" button (`mediation: 'optional'`,
  passing the typed email to `GET /auth/login/start?email=…`); on success it
  posts the assertion to `/auth/login/finish`, calls `getMe()` to set
  `currentUser`, and switches `currentView` to `"dashboard"`. A "Register"
  sub-form POSTs to `/auth/register/start` and shows a generic "check your
  email" confirmation. Inline error states cover deactivated (403), rate limit
  (429), registration closed (403), failed assertion (401), and network errors.
  `go.sstools.co` branding in the header.
- `web/src/lib/api.ts` — `loginStart`, `loginFinish`, `registerStart` helpers.
- Added `vitest` (`npm test` → `vitest run`) and
  `web/src/lib/webauthn.test.ts`.

The original AC said "calls `GET /auth/login/start` on mount to pre-load a
challenge"; this is realized as the on-mount conditional-mediation ceremony,
which both pre-loads the challenge and arms browser passkey autofill (the
purpose the AC cites). The explicit button issues a second, email-scoped
`login/start` so the server can return `allowCredentials`.

## Gotchas

- **Encoding contract.** The server (go-webauthn) serializes and parses every
  binary field as **unpadded base64url** (`base64.RawURLEncoding`, the
  `URLEncodedBase64` type). `btoa`/`atob` use *standard* base64 with `+`/`/`
  and `=` padding, so `bufferToBase64url` must translate the alphabet
  (`+`→`-`, `/`→`_`) and strip `=`. A mismatch here silently fails the
  ceremony — confirmed against `protocol/base64.go`, `credential.go`,
  `assertion.go`, and `authenticator.go` in go-webauthn v0.17.x.
- The finish JSON keys are `id`, `type`, `rawId`, `response.clientDataJSON`,
  `response.authenticatorData`, `response.signature`, `response.userHandle`
  (omit when empty) — exactly `protocol.CredentialAssertionResponse`.
- TypeScript 5.7's typed-array generics mean `Uint8Array` from a generic
  `ArrayBufferLike` is not assignable to the DOM `BufferSource`; backing the
  decode result with an explicit `new ArrayBuffer(len)` (typed
  `Uint8Array<ArrayBuffer>`) satisfies `svelte-check`.
