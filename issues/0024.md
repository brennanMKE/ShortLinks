## Description

Implement the URL filtering subsystem: load active rules from the DB (cached 60s), evaluate each rule's regex pattern against the destination URL at link creation, store denied links with the appropriate reason code, and provide admin CRUD + test routes.

## Acceptance Criteria

- [x] `internal/filters/` package with `LoadActiveRules(ctx, db) ([]Rule, error)` and `Evaluate(rules []Rule, url string) (reasonCode int, matched bool)` (Evaluate also returns the matched rule id)
- [x] Rules are cached in `internal/cache/` with 60-second TTL; cache is invalidated immediately on any rule create/update/delete
- [x] `Evaluate` uses `regexp` (not PostgreSQL); first matching rule wins; returns 0 and false if no rule matches
- [x] On match: `INSERT` link row with `active=false, denied_reason=<code>`; return `422 {"error":"url_denied","reason":<code>,"label":"<label>"}`. The `link.denied` audit entry is left as the #0025 seam (audit write path not yet built); the values are captured at the seam.
- [x] Denial reason codes: 0=none, 1=malware, 2=phishing, 3=spam, 4=adult_content, 5=policy_violation, 6=other
- [x] `GET /admin/url-filters`: list all rules; requires admin auth
- [x] `POST /admin/url-filters`: body `{pattern, reason_code, description}`; validates pattern compiles as Go regex; inserts rule; invalidates cache; returns `201`. `url_filter.created` audit left as #0025 seam.
- [x] `PATCH /admin/url-filters/{id}`: partial update; invalidates cache. `url_filter.updated` audit left as #0025 seam.
- [x] `DELETE /admin/url-filters/{id}`: deletes rule; invalidates cache. `url_filter.deleted` audit left as #0025 seam.
- [x] `POST /admin/url-filters/test`: body `{url}`; evaluates against all active rules; returns `{"matched": true, "reason_code": 2, "rule_id": 5}` or `{"matched": false}`
- [x] `go test ./internal/filters/` covers: match first rule, no match, invalid regex rejected; cache invalidation covered in `internal/cache/` and via the admin integration test

## Implementation

**Engine (`internal/filters/filters.go`).** Denial-reason constants `ReasonNone..ReasonOther` (0..6) pinned to the PRD table, a `reasonLabels` map + `ReasonLabel(code)` (unknown â†’ "Other"), and `ValidReasonCode` (1..6; 0 rejected since a rule always denies). `Rule{ID, Pattern, ReasonCode, compiled}`. `LoadActiveRules(ctx, Querier)` reads active rows in id order. `CompileRules(rules, logger)` compiles each pattern once, **skipping + logging** an uncompilable one so it cannot break the set. `Evaluate(rules, url)` returns the first match's `(reasonCode, ruleID, true)` or `(0, 0, false)`, preferring the pre-compiled regex and compiling on the fly (skip-on-error) for raw rules.

**Store (`internal/filters/store.go`).** `filters.Store` over the pool: `LoadActive`, `List`, `Create`, `Get`, `Update` (dynamic SET), `Delete`, `ErrRuleNotFound`.

**Rule cache (`internal/cache/rules.go`).** `RuleCache` holds one mutex-guarded snapshot of the active compiled rules with `FilterRuleTTL = 60s`. `Rules(ctx)` lazily loads/refreshes; a loader error keeps the stale snapshot. `Invalidate()` drops it so a mutation takes effect at once.

**Denied insert (`internal/links/store.go`).** `CreateDeniedLink(ctx, in, reasonCode, genKey)` mints a unique generated key and inserts `active=false, denied_reason=code`. Denied rows are excluded from the dedup index, so each blocked submission inserts a fresh row (PRD per-attempt count).

**Create wiring (`internal/handlers/links.go`).** The `#0024` seam now runs FIRST (before dedup) in BOTH the generated-key and custom-alias branches: load rules via the `ruleProvider` (nil â†’ filtering disabled), `filters.Evaluate`, and on a match call `CreateDeniedLink`, leave the `link.denied` #0025 audit seam (capturing `{destination_url, reason_code, reason_label, matched_rule_id}`), and return **422** `{error:"url_denied", reason, label}`. SSE (#0026) is intentionally NOT fired for a denial.

**Admin CRUD (`internal/handlers/url_filters.go`).** `URLFiltersHandler` serves `GET/POST/PATCH/DELETE /admin/url-filters` + `POST /admin/url-filters/test` behind `RequireSession`+`RequireAdmin`. Create/Patch validate the regex compiles and `reason_code` is 1..6; every mutation calls `RuleCache.Invalidate()` and leaves the `url_filter.created/updated/deleted` #0025 audit seams. `created_by` is the ctx admin id. Routes wired in `cmd/shortlinks/main.go`, where the rule cache loader compiles via the filters store.

## Gotchas

- `url_filter_rules` is NOT in the credentials suite's `TRUNCATE` list, so the integration tests truncate it explicitly (`filterTestPool`/`clearFilterRules`) before and after each run to stay isolated and leave the DB clean.
- The denial path mints a generated key for the denied row even when the client supplied a custom alias â€” a denied submission never consumes the requested alias, and denied rows are kept out of the dedup index so repeats accrue rather than reactivate.
- `ValidReasonCode` deliberately rejects `0`: a filter rule always denies, so a `reason_code` of "none" is a client error (400), not a valid rule.
