## Description

Implement the URL filtering subsystem: load active rules from the DB (cached 60s), evaluate each rule's regex pattern against the destination URL at link creation, store denied links with the appropriate reason code, and provide admin CRUD + test routes.

## Acceptance Criteria

- [ ] `internal/filters/` package with `LoadActiveRules(ctx, db) ([]Rule, error)` and `Evaluate(rules []Rule, url string) (reasonCode int, matched bool)`
- [ ] Rules are cached in `internal/cache/` with 60-second TTL; cache is invalidated immediately on any rule create/update/delete
- [ ] `Evaluate` uses `regexp.MatchString` (not PostgreSQL); first matching rule wins; returns 0 and false if no rule matches
- [ ] On match: `INSERT` link row with `active=false, denied_reason=<code>`; write `link.denied` audit entry with `{destination_url, reason_code, reason_label, matched_rule_id}`; return `422 {"error":"url_denied","reason":<code>,"label":"<label>"}`
- [ ] Denial reason codes: 0=none, 1=malware, 2=phishing, 3=spam, 4=adult_content, 5=policy_violation, 6=other
- [ ] `GET /admin/url-filters`: list all rules; requires admin auth
- [ ] `POST /admin/url-filters`: body `{pattern, reason_code, description}`; validates pattern compiles as Go regex; inserts rule; invalidates cache; writes `url_filter.created` audit entry; returns `201`
- [ ] `PATCH /admin/url-filters/{id}`: partial update; invalidates cache; writes `url_filter.updated` audit entry
- [ ] `DELETE /admin/url-filters/{id}`: deletes rule; invalidates cache; writes `url_filter.deleted` audit entry
- [ ] `POST /admin/url-filters/test`: body `{url}`; evaluates against all active rules; returns `{"matched": true, "reason_code": 2, "rule_id": 5}` or `{"matched": false}`
- [ ] `go test ./internal/filters/` covers: match first rule, no match, invalid regex rejected, cache invalidation
