| | |
|---|---|

## Description

Write `docs/configuration.md` documenting every configuration value the service reads, sourced from `internal/config` and `.env.example`.

## Scope

- A table of every variable: `PORT`, `BASE_URL`, `DATABASE_URL`, `WEBAUTHN_RP_ID`, `WEBAUTHN_RP_ORIGIN`, `SESSION_SECRET`, `SES_SMTP_*`, `EMAIL_FROM`, `CACHE_MAX_COST`, `CACHE_TTL_SECONDS`, `ADMIN_EMAIL` — with required/optional, default, and purpose.
- How config is loaded (the loader, required-variable validation) and where it lives in dev (`.env`) vs production (`/etc/shortlinks/config.env`).
- Secrets handling (what must never be committed); reference `.env.example`.

## Acceptance criteria

- [ ] `docs/configuration.md` lists all variables matching `internal/config` + `.env.example`, with required/optional/defaults correct
- [ ] Documents the loader behavior and dev-vs-prod config file locations
