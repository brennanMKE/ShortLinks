## Description

Implement the short-link key generation logic in `internal/links/`. Keys are 6-character base-62 strings. Generation must handle collisions by retrying up to 5 times.

- [x] Generates a key, checks for uniqueness, retries up to 5 times on collision, returns an error after 5 failures — implemented as `GenerateUniqueKey(exists func(key string) (bool, error)) (string, error)` so the collision check is injectable and DB-agnostic

## Implementation

Added `internal/links/keygen.go` in package `links`:

- `GenerateKey() (string, error)` — produces a 6-character base-62 key
  (`[a-z0-9A-Z]`, 62 chars) using `crypto/rand`. Modulo bias is avoided via
  rejection sampling: random bytes `>= 248` (the largest multiple of 62 that
  fits in a byte) are discarded and re-drawn, so all 62 symbols are equally
  likely. Constants `KeyLength = 6` and the alphabet are package-level.
- `GenerateUniqueKey(exists func(key string) (bool, error)) (string, error)` —
  generates a candidate, calls `exists` to test uniqueness, and retries up to
  `maxKeyAttempts = 5` times. Returns the exported sentinel `ErrKeyCollision`
  after 5 consecutive collisions, propagates (wrapped) any error from `exists`,
  and rejects a nil callback. The callback abstraction keeps the logic fully
  unit-testable without a database; the DB-backed caller (#0022) will pass a
  closure that runs the `links.key` uniqueness query.

The DB-coupled `CreateLink` / `CreateLinkWithAlias` wrappers are intentionally
deferred to #0022 (no DB available in this environment); the reusable,
DB-agnostic core they depend on is delivered here.

## Gotchas

- Used `crypto/rand` (not `math/rand`) for unpredictability, with rejection
  sampling to eliminate modulo bias across the 62-char alphabet.
- The DB collision check is abstracted behind the `exists` callback so this
  package compiles and is tested with no PostgreSQL dependency.
