## Description

Bootstrap the Svelte 5 + Vite + TypeScript project under `web/`. Set up the shared stores, API client, type definitions, and dev-server proxy. This issue covers project structure only — individual views are in later issues.

## Acceptance Criteria

- [ ] `web/package.json` with dependencies: `svelte@5`, `vite`, `@sveltejs/vite-plugin-svelte`, `typescript`
- [ ] `web/vite.config.ts` with dev proxy:
  ```typescript
  proxy: {
    '/api': 'http://localhost:8080',
    '/auth': 'http://localhost:8080',
    '/u': 'http://localhost:8080',
  }
  ```
- [ ] `web/src/lib/stores.ts` defines writable stores: `currentView` (type `"login" | "dashboard" | "link-detail" | "account" | "admin"`), `currentUser`, `links`
- [ ] `web/src/lib/api.ts` defines typed fetch wrappers for all API endpoints (links CRUD, /api/me, auth routes)
- [ ] `web/src/lib/types.ts` defines TypeScript interfaces for `Link`, `User`, `Credential`, `AuditEntry`, `FilterRule`, `ClickStats`
- [ ] `web/src/App.svelte` renders the correct view component based on `currentView` store; on mount calls `GET /api/me` to set `currentUser` and determine initial view (login if 401, dashboard otherwise)
- [ ] `web/src/main.ts` mounts `App.svelte`
- [ ] `npm run dev` starts Vite dev server without errors
- [ ] `npm run build` produces output in `web/dist/`
- [ ] `.gitignore` includes `web/node_modules/` and `web/dist/`
