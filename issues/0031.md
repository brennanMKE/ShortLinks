## Description

Bootstrap the Svelte 5 + Vite + TypeScript project under `web/`. Set up the shared stores, API client, type definitions, and dev-server proxy. This issue covers project structure only — individual views are in later issues.

## Acceptance Criteria

- [ ] `web/package.json` with dependencies: `svelte@5`, `vite`, `@sveltejs/vite-plugin-svelte`, `typescript`
- [ ] `web/vite.config.ts` with dev proxy:
  ```typescript
  proxy: {
    '/api': 'http://localhost:8080',
    '/auth': 'http://localhost:8080',
    '/u': 'http://localhost:8080',
  }
  ```
- [ ] `web/src/lib/stores.ts` defines writable stores: `currentView` (type `"login" | "dashboard" | "link-detail" | "account" | "admin"`), `currentUser`, `links`
- [ ] `web/src/lib/api.ts` defines typed fetch wrappers for all API endpoints (links CRUD, /api/me, auth routes)
- [ ] `web/src/lib/types.ts` defines TypeScript interfaces for `Link`, `User`, `Credential`, `AuditEntry`, `FilterRule`, `ClickStats`
- [ ] `web/src/App.svelte` renders the correct view component based on `currentView` store; on mount calls `GET /api/me` to set `currentUser` and determine initial view (login if 401, dashboard otherwise)
- [ ] `web/src/main.ts` mounts `App.svelte`
- [ ] `npm run dev` starts Vite dev server without errors
- [ ] `npm run build` produces output in `web/dist/`
- [ ] `.gitignore` includes `web/node_modules/` and `web/dist/`

## Implementation

Bootstrapped the `web/` SPA — the frontend foundation for #0032–#0037 (views)
and #0038 (`go:embed`). Stack: **Svelte 5.43** + **Vite 6.4** + **TypeScript
5.9**, type-checked with `svelte-check 4.x`. No client-side router — navigation
is a write to a `currentView` store.

Files created:

- `package.json` — scripts `dev`, `build`, `preview`, and `check`
  (svelte-check); devDeps `svelte@^5`, `vite@^6`,
  `@sveltejs/vite-plugin-svelte`, `typescript`, `svelte-check`,
  `@tsconfig/svelte`. `package-lock.json` committed for reproducible installs.
- `vite.config.ts` — Svelte plugin + dev `server.proxy` for `/api`, `/auth`,
  `/u` → `http://localhost:8080`; build `outDir: dist`.
- `tsconfig.json`, `svelte.config.js` (vitePreprocess), `index.html`,
  `src/main.ts` (Svelte 5 `mount(App, …)`), `src/vite-env.d.ts`.
- `src/lib/stores.ts` — `currentView` (`"login" | "dashboard" | "link-detail"
  | "account" | "admin"`, default `"login"`), `currentUser` (User | null),
  `links` (Link[]), `selectedLinkKey`.
- `src/lib/api.ts` — typed fetch wrapper: `apiGet`/`apiPost`/`apiPatch`/
  `apiDelete` (same-origin, `credentials:'include'`, JSON headers, JSON parse,
  throws typed `ApiError` carrying `status` + parsed body on non-2xx). Endpoint
  helpers: `getMe`, `listLinks`, `getLink`, `createLink`, `updateLink`,
  `deactivateLink`, `logout`, credential + admin helpers.
- `src/lib/types.ts` — interfaces matching the Go JSON exactly (snake_case):
  `User`, `Link`/`LinkList`/`LinkDetail`, `ClickStats`/`UTMBucket`,
  `Credential`, `AuditEntry`, `FilterRule`, `AdminUser`, `Setting`. Field names
  cross-checked against `internal/handlers/{me,links,credentials,audit,
  url_filters,users,settings}.go` and `internal/clicks/stats.go`.
- `src/App.svelte` — renders the active view from `currentView`; on mount calls
  `GET /api/me`, sets `currentUser`, lands on `dashboard` (or `login` on 401).
- `src/views/{Login,Dashboard,LinkDetail,Account,Admin}.svelte` — trivial
  placeholders (heading only); full implementations land in #0032–#0037.

Removed the now-unneeded `web/.gitkeep`.

## Gotchas

- `node_modules/` and `dist/` are gitignored (root `.gitignore`) — only source
  + `package-lock.json` are tracked. Run `npm install` after clone.
- Svelte 5 uses the imperative `mount()` API in `main.ts` (not `new App({…})`).
- API type interfaces are deliberately snake_case to match the Go JSON tags
  one-to-one — do not camelCase them.
