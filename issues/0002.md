## Description

Create the one-time database initialisation scripts that a new server operator runs before migrations. These are separate from the migration files — they handle the PostgreSQL user account and database creation that `golang-migrate` cannot do for itself.

## Acceptance Criteria

- [ ] `scripts/db/create.sql` creates the database and application role:
  - `CREATE ROLE shortlinks LOGIN PASSWORD '...'`
  - `CREATE DATABASE shortlinks OWNER shortlinks`
  - `GRANT ALL PRIVILEGES ON DATABASE shortlinks TO shortlinks`
- [ ] `scripts/db/drop.sql` safely tears down the database and role for development resets (`DROP DATABASE IF EXISTS shortlinks; DROP ROLE IF EXISTS shortlinks`)
- [ ] `scripts/db/README.md` documents how to run both scripts (`psql -U postgres -f scripts/db/create.sql`) and notes that the password must be set before running
- [ ] Scripts are idempotent where possible (use `IF NOT EXISTS` / `IF EXISTS`)
- [ ] `DATABASE_URL` format in `.env.example` matches the role and database name defined here

## Notes

These scripts are run once by an admin user (`postgres` superuser or equivalent). They do not replace migrations — migrations handle schema. Password in the script is a placeholder; operators substitute their own before running.
