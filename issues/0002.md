## Description

Create the one-time database initialisation scripts that a new server operator runs before migrations. These are separate from the migration files — they handle the PostgreSQL user account and database creation that `golang-migrate` cannot do for itself.

## Acceptance Criteria

- [ ] `scripts/db/create.sql` creates the database and application role:
  - `CREATE ROLE shortlinks LOGIN PASSWORD '...'`
  - `CREATE DATABASE shortlinks OWNER shortlinks`
  - `GRANT ALL PRIVILEGES ON DATABASE shortlinks TO shortlinks`
- [ ] `scripts/db/drop.sql` safely tears down the database and role for development resets (`DROP DATABASE IF EXISTS shortlinks; DROP ROLE IF EXISTS shortlinks`)
- [ ] `scripts/db/README.md` documents how to run both scripts (`psql -U postgres -f scripts/db/create.sql`) and notes that the password must be set before running
- [ ] Scripts are idempotent where possible (use `IF NOT EXISTS` / `IF EXISTS`)
- [ ] `DATABASE_URL` format in `.env.example` matches the role and database name defined here

## Notes

These scripts are run once by an admin user (`postgres` superuser or equivalent). They do not replace migrations — migrations handle schema. Password in the script is a placeholder; operators substitute their own before running.

## Implementation

Created `scripts/db/create.sql` and `scripts/db/drop.sql` for one-time database bootstrap. `create.sql` creates the `shortlinks` login role, the `shortlinks` database owned by that role, and grants all privileges; role creation is wrapped in a `DO`/`pg_roles` guard so it is idempotent, with a comment noting that `CREATE DATABASE` cannot be guarded (PostgreSQL has no `IF NOT EXISTS` for databases). `drop.sql` tears both down with `DROP DATABASE IF EXISTS` / `DROP ROLE IF EXISTS` for safe, repeatable development resets. Added `scripts/db/README.md` documenting the run commands (`psql -U postgres -f ...`) and the requirement to replace the placeholder password first, and updated `.env.example` so `DATABASE_URL` uses the `shortlinks` role and database name. The password is the clearly-labelled placeholder `CHANGE_ME_IN_PRODUCTION`, never a real secret.

## Gotchas

PostgreSQL has no `CREATE DATABASE IF NOT EXISTS`, and `CREATE DATABASE` cannot run inside a transaction or `DO` block. So `create.sql` is only partially idempotent — re-running it on an existing database raises a harmless "database already exists" error, which is documented inline and in the README.
