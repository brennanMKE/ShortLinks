All deployment documentation was written for Ubuntu 22.04 LTS. The target EC2 OS is Amazon Linux 2023 (AL2023), which uses `dnf` instead of `apt`, `httpd` instead of `apache2`, and has different Apache config conventions. Revise every affected `.md` file so the commands work on AL2023 without translation.

## Root cause

All docs were written against Ubuntu 22.04 (`apt`, `apache2`, `a2enmod`, `a2ensite`). AL2023 uses `dnf`, names the Apache package and service `httpd`, includes proxy/SSL modules by default, and uses a drop-in `/etc/httpd/conf.d/` directory instead of `sites-available` + `a2ensite`. PostgreSQL on AL2023 also requires an explicit `postgresql-setup --initdb` before the first service start.

## Fix

Updated `DEPLOYMENT.md` (Prerequisites, Database, Apache, and TLS sections) and `deploy/apache/README.md`. `deploy/systemd/README.md` and `docs/email_setup.md` contained no OS-specific commands and needed no changes.
