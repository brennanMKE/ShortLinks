## Description

Implement the redirect handler — the hottest path in the system. Every public click hits this endpoint. It must check the cache first, fall back to the DB, record the click asynchronously, merge UTM params, and return a 302.

## Acceptance Criteria

- [ ] `GET /u/{key}` handler implemented in `internal/handlers/`
- [ ] Check cache → on miss query DB → populate cache
- [ ] `active = false` or key not found → `404`
- [ ] `expires_at` in the past → `410 Gone`
- [ ] Click recorded asynchronously (goroutine, non-blocking) with: `link_id`, `clicked_at`, `ip_address`, `user_agent`, `referer`, all 5 `utm_*` params from query string
- [ ] Inbound `utm_*` query params merged onto `destination_url` before redirect (existing params on destination preserved; inbound params override on conflict)
- [ ] Returns `302 Found` with `Location` header
- [ ] `internal/clicks/` package handles DB insert for click rows
- [ ] `go test ./internal/handlers/` covers: cache hit, cache miss, 404, 410, UTM merge, click recorded

## Implementation

Added `internal/handlers/redirect.go` implementing the PRD's 7-step Redirect
Behavior. The handler is built for unit testability via two injected
interfaces, so it needs no real database:

- `LinkResolver.Resolve(ctx, key) (cache.CachedLink, bool, error)` abstracts the
  cache→DB lookup. The cache+DB-backed implementation is wired later once the DB
  layer exists (deps #0006–#0008); this issue depends only on the interface and
  tests use a fake. Negative cache entries are reported as `found=false` so the
  handler treats them as absent. A non-nil error yields `500`.
- `ClickRecorder.RecordClick(ClickInfo)` records the click. The handler snapshots
  request metadata, then calls it in a goroutine (`go h.recorder.RecordClick(info)`)
  so logging never blocks the redirect.

Flow: extract `{key}` via Go 1.22 `http.ServeMux` pattern routing
(`GET /u/{key}`, `r.PathValue("key")`); resolve; `404` on miss/negative/inactive;
`410` when `ExpiresAt` is non-nil and not in the future; record click async with
IP / User-Agent / Referer / 5 `utm_*` values; merge inbound `utm_*` onto the
destination with `net/url`; `302 Found` with the merged `Location`.

Merge precedence: the destination's own query params are preserved; inbound
`utm_*` values override the destination's value for the same `utm_*` key (only
the five known UTM keys are forwarded). Client IP prefers the first
`X-Forwarded-For` hop (Apache reverse proxy), falling back to `RemoteAddr` with
the port stripped. A clock is injectable for deterministic expiry tests.

Note: the `internal/clicks/` DB-insert package (a separate acceptance item) is
deferred to its dependency issues; this handler is decoupled from it via the
`ClickRecorder` interface.

## Verification

Commands run and observed results:

```
go build ./...                              # BUILD OK (whole module compiles)
go test ./...                               # ok  github.com/brennanMKE/ShortLinks/internal/handlers  0.863s
go vet ./internal/handlers/                 # VET OK
go test -race -count=3 -v ./internal/handlers/   # PASS — ok ... 2.781s
```

The `-race -count=3` run executed all 10 tests three times each with the race
detector enabled and reported no data races (the async click goroutine is
clean). Tests:

- `TestActiveLinkRedirects302` — active link → 302 with correct Location
- `TestUnknownKeyReturns404` — unknown key → 404, no click
- `TestNegativeCacheEntryReturns404` — negative cache entry → 404, no click
- `TestInactiveLinkReturns404` — `Active=false` → 404, no click
- `TestExpiredLinkReturns410` — past `ExpiresAt` → 410, no click
- `TestFutureExpiryStillRedirects` — future `ExpiresAt` → 302
- `TestUTMParamsMergedAndRecorded` — all 5 `utm_*` merged into Location and
  captured in the recorded click; metadata (UA, Referer, XFF IP) captured
- `TestDestinationQueryParamsPreservedOnMerge` — pre-existing dest params kept;
  inbound `utm_*` overrides the destination's
- `TestResolverErrorReturns500` — resolver error → 500, no click
- `TestRemoteAddrUsedWhenNoForwardedFor` — IP from `RemoteAddr`, port stripped

The async click is observed deterministically via a buffered channel + a
`select` with timeout (no `time.Sleep`).

## Gotchas

- Ristretto `Set` is asynchronous, but the handler depends on `LinkResolver`,
  not the concrete cache, so tests avoid that entirely with a fake.
- The click goroutine must capture a value snapshot before launch — the
  `*http.Request` must not be read after the response is written.
