## Description

Create the Phase 1 database migrations using `golang-migrate/migrate` naming conventions. Migrations must run in order and be reversible. The `users` table must be created before `links` due to the foreign key.

## Acceptance Criteria

- [ ] `migrations/0001_create_users.up.sql` ‚Äî creates `users` table with all columns from PRD: `id`, `email`, `is_admin`, `active`, `created_at`, `last_login_at`
- [ ] `migrations/0002_create_links.up.sql` ‚Äî creates `links` table: `id`, `user_id` (FK ‚Üí users), `key`, `destination_url`, `title`, `created_at`, `expires_at`, `active`, `denied_reason`
- [ ] `migrations/0003_create_clicks.up.sql` ‚Äî creates `clicks` table with all UTM columns
- [ ] `migrations/0004_create_indexes.up.sql` ‚Äî creates all Phase 1 indexes: `links(key)` (covered by UNIQUE), `links(user_id)`, `links(user_id, destination_url) WHERE denied_reason = 0`, `links(denied_reason) WHERE denied_reason > 0`, `clicks(link_id)`, `clicks(clicked_at)`
- [ ] Corresponding `.down.sql` files drop tables/indexes in reverse order
- [ ] `migrate -path migrations -database $DATABASE_URL up` runs cleanly against a fresh database
- [ ] `migrate -path migrations -database $DATABASE_URL down` reverses all migrations cleanly
