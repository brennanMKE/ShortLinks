## Description

Implement the Dashboard view (`web/src/views/Dashboard.svelte`). Shows a link creation form and a live-updating list of the user's recent short links. SSE integration is a separate issue (#0034).

## Acceptance Criteria

- [ ] On mount, loads link list via `GET /api/links` and populates `links` Svelte store
- [ ] Link creation form with fields: `destination_url` (required), `title` (optional), `key` (optional custom alias); submits to `POST /api/links`
- [ ] On successful creation shows the generated short URL (`https://go.sstools.co/u/{key}`) with a copy-to-clipboard button
- [ ] On `duplicate: true` response, shows inline notice: "This URL was already shortened — here's your existing link"
- [ ] On `422` (url_denied) response, shows the denial reason label inline
- [ ] Link list renders each link with: short URL, destination domain, title, created_at, click count, active status; clicking a link navigates to `"link-detail"` view and sets a `selectedLink` store
- [ ] "Sign out" action calls `POST /auth/logout` and navigates to `"login"`
- [ ] Navigation tabs visible for: Dashboard, Account; Admin tab visible only when `currentUser.is_admin === true`
- [ ] Component is shown when `currentView === "dashboard"`

## Notes

Dependencies: #0022 (link CRUD API), #0031 (stores, API client), #0034 (SSE integration added on top).

## Implementation

Replaced the placeholder `web/src/views/Dashboard.svelte` with the full view (Svelte 5 runes, matching `Login.svelte`'s style/error handling):

- **Create form** — `destination_url` (required, client-validated as http(s) via `isValidHttpUrl`, mirroring the server's `validDestinationURL`), optional `title`, optional custom alias sent as `key` (the canonical field the POST handler reads; it also accepts `custom_key`/`alias` synonyms), and optional expiry via `<input type="datetime-local">` converted to an RFC 3339 instant for `expires_at`. Submits via `createLink`.
- **Create response handling** — mapped by `noticeForCreated` / `noticeForError` (in `lib/links.ts`): a fresh link shows the generated `https://go.sstools.co/u/{key}` short URL with a copy button; `duplicate: true` shows the "This URL was already shortened — here's your existing link" notice while still surfacing the returned link; a **422** `url_denied` shows the server's `label` (falling back to the PRD code→label table); **409** renders an inline "alias taken" error on the custom-alias field; **400** an inline bad-URL error on the destination field; **401** redirects to login.
- **List** — loaded on mount via `listLinks(page, per_page)` into the shared `links` store. Columns: short URL (`/u/{key}`) + copy button, destination domain, title, click count, status badge (active/denied/inactive from `linkStatus`), created date. Clicking a row sets `selectedLinkKey` + `currentView="link-detail"` (#0035). A row-level Deactivate action calls `deactivateLink` and flips the row's `active` in the store. Pagination via Previous/Next using the server's `page`/`per_page`/`total`.
- **Header** — Dashboard/Account tabs, an Admin tab gated on `currentUser.is_admin`, and a Sign out button calling `logout()` then returning to login.
- **#0034 SSE seam** — left as a clearly-labelled comment block inside `onMount` in `Dashboard.svelte` describing the `new EventSource('/api/events')` subscription, the `link.created` prepend-into-`links`-store, and the `eventSource.close()` on unmount. The list is loaded purely via REST here so the dashboard works without SSE; the shared `links` store is the prepend target #0034 will use.

Pure, DOM-free logic was factored into `web/src/lib/links.ts` (`shortUrl`, `isValidHttpUrl`, `deniedReasonLabel`, `noticeForCreated`, `noticeForError`, `linkStatus`, `destinationDomain`) and unit-tested.

## Gotchas

- The displayed short URL is fixed to the production base `https://go.sstools.co/u/{key}` (built from the link's `key`), intentionally NOT the backend's dev `BASE_URL` — a copied localhost link is useless to share. This is baked into `SHORT_URL_BASE`.
- The POST handler reads the custom alias from `key` (canonical), with `custom_key`/`alias` accepted as synonyms; the client sends `key` via the existing `CreateLinkInput`.
- Svelte's `svelte-check` does not narrow a `$state` union across an inline arrow event handler, so `notice.link.key` inside an `onclick` triggered a "possibly null / property missing" error. Captured the value into a block-local `{@const resultKey = notice.link.key}` after the `kind` guard to satisfy the checker.
