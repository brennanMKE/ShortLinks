## Implementation

Ran `go mod init github.com/brennanMKE/ShortLinks` (Go 1.26.3). Created the full
`internal/*` package tree, each with a one-line package-doc file (`doc.go`) so the
directories are valid, committable Go packages that `go build ./...` compiles —
git cannot track empty directories, and an empty dir is not a Go package. Added a
minimal `cmd/shortlinks/main.go` that prints `shortlinks <version>` and exits.
Created `.env.example` mirroring the PRD Configuration section verbatim.
`migrations/` and `web/` are kept in git via `.gitkeep` (both filled by later issues).

## Gotchas

The pre-existing `.gitignore` ignored the bare pattern `shortlinks`, which git
matches at *any* depth — so it silently excluded the `cmd/shortlinks/` source
directory, not just the built root binary. `git add cmd/shortlinks/main.go` failed
with "paths are ignored". Fixed by anchoring the pattern to the repo root as
`/shortlinks`. Watch for the same trap with any other unanchored binary-name ignore.
