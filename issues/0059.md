## Description

Provide a single command (e.g. `make dev` or `scripts/dev.sh`) that builds and runs the website locally on this Mac — Go server + SPA — using the JSON dev store (#0057) and dev auth (#0058), with **no systemd, no PostgreSQL, and no migrations**. This is what lets us run the app to check web pages directly.

## Motivation

The current run paths assume Postgres + migrations + a systemd service (production), or a manual multi-step dev setup. A one-shot local runner closes the loop for fast iteration and verification. (Part of [#0056](0056.md).)

## Proposed approach

- A `make dev` target / `scripts/dev.sh` that sets the dev env (`STORAGE=json` / no `DATABASE_URL`, `DEV_AUTH=true`, localhost `BASE_URL`), runs the Go server (`go run ./cmd/shortlinks serve`), and either serves the embedded SPA or runs the Vite dev server (`web/`, with the existing `/api` proxy) for hot reload. Print the localhost URL.
- No systemd, no `migrate`, no Postgres. Idempotent, safe to re-run, clean shutdown.
- Offer a built-SPA variant too (`npm run build` + `go run`) so what's tested matches the production embedding.
- Document it in `README.md` (Quick Start) and/or a new `docs/dev.md`.

## Acceptance criteria

- [x] One command builds and serves the app on localhost with no Postgres / systemd / migrations
- [x] It lands on the dashboard (via #0058) backed by the JSON store (#0057)
- [x] Documented; clean start/stop
- [x] Does not alter the production build/deploy path (`scripts/deploy.sh`, systemd unit)

## Implementation

Added `scripts/dev.sh` (+ a `Makefile` with `dev`/`dev-built`/`build`/`test` targets). Default mode sets dev env (`STORAGE=json`, `BASE_URL`/`WEBAUTHN_RP_*`/`SESSION_SECRET`/`ADMIN_EMAIL`/`PORT`, all overridable), starts the Go API on `:8080` (in-memory dev store #0057 + dev auto-login #0058) in the background, and runs Vite on `:5173` in the foreground (proxying `/api /auth /account /admin /u` to `:8080`); an `EXIT/INT/TERM` trap kills the Go process on Ctrl-C. `--built` mode runs `npm run build` then serves the embedded SPA at `:8080` to match the production embed. No `migrate`/`psql`/`systemctl` anywhere. `docs/dev.md` documents both modes; the `README.md` Quick Start now leads with the one command, with the Postgres/production steps preserved in a collapsed block (not deleted). Built on `issue/0059` (Sonnet implement, Opus review → APPROVE), squash-merged to `main` as `9986c8e`.

## Verification

`bash -n scripts/dev.sh` clean. Running the orchestrated pieces (re-run by the reviewer): Go API with `STORAGE=json` → `/health` `200`, `/api/me` `200 {"id":1,"email":"admin@localhost","is_admin":true}`; through the Vite `:5173` proxy `/api/me` also `200` mock admin; `localhost:5173/` serves the Vite index — all with no Postgres. The production path (`scripts/deploy.sh`, systemd unit, production build) is not in the diff. **Known gap for a human eyeball:** `--built` mode wasn't run headlessly (it produces a `web/dist` build that must not be committed) — do one manual `./scripts/dev.sh --built` to confirm the embedded path (favicon, SPA deep links).

## Gotchas

- The dev session cookie is `Secure` (reused from real auth), so a stale `curl` jar over http can 401 on a second-hand jar; a real browser treats `localhost` as a secure context and the per-request auto-login (#0058) authenticates regardless. Non-issue in a browser.
