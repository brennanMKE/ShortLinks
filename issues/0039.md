# 0039 — Concurrent `go test ./...` corrupts shared test DB across integration packages

| | |
|---|---|
| **Status** | open |
| **Module** | db / auth / handlers / middleware |
| **Platform** | All |
| **First seen** | 2026-05-25 |

## Description

The integration tests in `internal/auth`, `internal/handlers`, `internal/middleware`,
`internal/db`, and `cmd/shortlinks` all connect to the same `TEST_DATABASE_URL`
(`shortlinks_test`) and `TRUNCATE` the shared tables (`users`, `passkey_credentials`,
`sessions`, etc.) in their setup helpers. `go test` runs package test binaries concurrently by
default, so while one package has seeded rows and is mid-assertion, another package's `TRUNCATE`
wipes them — producing foreign-key violations and false failures.

`go test ./...` (the verify command documented in `issues/Issues.md`) is therefore unreliable.
The only current workaround is `go test -p 1 ./...`, which serializes packages.

This was discovered while resolving #0019.

## Steps to reproduce

1. Ensure `shortlinks_test` is migrated and `TEST_DATABASE_URL` is exported.
2. Run `go test ./...` (no `-p 1`).
3. Observe intermittent failures such as `passkey_credentials rows = 0, want 1`,
   `insert or update on table "passkey_credentials" violates foreign key constraint`, and
   `first user should be promoted to admin`.

## Expected

`go test ./...` passes reliably without requiring `-p 1`.

## Actual

Concurrent package execution against one shared mutable database causes cross-package data
corruption and non-deterministic failures.

## Acceptance Criteria

- [ ] Integration test packages no longer interfere when run concurrently.
- [ ] `go test ./...` (no `-p 1`) passes reliably across repeated runs.
- [ ] The fix does not require every developer to remember a special flag.

## Notes

Likely fix: serialize DB access across packages with a PostgreSQL session-level advisory lock
acquired in a shared test helper invoked from each package's `TestMain`, so only one
DB-touching package runs at a time even under `go test`'s default concurrency. Affected
packages: `internal/auth`, `internal/handlers`, `internal/middleware`, `internal/db`,
`cmd/shortlinks`. None currently define `TestMain`.
