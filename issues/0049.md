# 0049 — Add visualizations (charts) for short-link click metrics

| | |
|---|---|
| **Status** | open |
| **Module** | web |
| **Platform** | All |
| **First seen** | 2026-06-18 |

## Description

Improve the short-link metrics display with actual visualizations — a clicks-over-time chart and visual breakdowns of the UTM dimensions — instead of today's text-only sorted lists. `LinkDetail.svelte` currently renders total clicks and per-UTM counts as plain lists; the backend stats in `internal/clicks/stats.go` already provide the data to chart.

## Motivation

The data to make metrics legible at a glance already exists but is presented as text. A time-series of clicks and proportional breakdowns (by source/medium/campaign) make trends and top performers obvious in a way ranked lists do not.

## Proposed approach

- Add a clicks-over-time series (line or bar) and proportional breakdowns (bar/donut) for the UTM dimensions on `LinkDetail`.
- **Charting library decision is part of this issue.** The SPA is embedded in the Go binary, so bundle size matters — prefer a lightweight option (e.g. hand-rolled inline SVG for these simple charts, or a small dependency) over a heavyweight suite. Document the choice and its gzipped cost; verify the production build size impact.
- Confirm whether the stats API exposes (or needs to expose) a time bucket for clicks-over-time — `internal/clicks/stats.go` may need a new query/endpoint for time-bucketed counts. Flag any backend change clearly; if it grows large, carve it into a `clicks`-module follow-up issue.
- Keep any data-shaping logic as pure, testable functions in `web/src/lib/`.
- Ensure charts are accessible (text alternative / data still available to screen readers) and render sensibly with empty or single-point data.

## Acceptance criteria

- [ ] `LinkDetail` shows a clicks-over-time chart and visual UTM breakdowns
- [ ] Charting approach chosen, justified, and its bundle-size impact recorded
- [ ] Any new backend stats endpoint (if needed) is implemented and tested in `internal/clicks`
- [ ] Empty-stats and single-data-point cases render gracefully (no broken axes)
- [ ] Charts have an accessible fallback
- [ ] `cd web && npm run build` and `npm run check` clean; Go tests pass if backend touched
